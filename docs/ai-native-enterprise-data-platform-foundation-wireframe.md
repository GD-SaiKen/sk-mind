## 1. 原型设计目标

基础平台页面原型的目标是支撑以下工作：

- 让平台管理员能看到基础平台整体运行状态。
- 让数据负责人能登记数据源、跟踪接入任务、维护字段说明和处理质量问题。
- 让业务用户能通过数据目录理解平台中已有数据。
- 让语义建模人员能维护对象、属性、关系、映射和对象行动策略。
- 让审计人员能查看数据访问和 Agent 调用记录。
- 让 Agent 使用者能在权限范围内查询数据，并看到来源、质量和限制说明。

原型重点不是展示复杂分析能力，而是把“数据进入平台并被可信使用”的基础链路跑通：

```text
数据源
→ 接入任务
→ 数据表
→ 数据目录
→ 数据质量
→ 权限控制
→ 语义映射
→ 关系图谱
→ Agent 数据服务
```

## 2. 页面设计原则

基础平台是企业数据治理和运营工具，页面应遵循以下原则：

| 原则 | 说明 |
| --- | --- |
| 状态优先 | 列表页优先展示状态、负责人、更新时间和异常 |
| 来源可见 | 数据表、字段、质量、语义映射和 Agent 回答都应能追踪来源 |
| 操作可控 | 高风险操作需要确认，Agent 操作需要权限和审计 |
| 信息密度适中 | 面向运营和治理场景，页面应紧凑、可筛选、可批量处理 |
| 异常可处理 | 不只展示失败，还要展示原因、影响范围和下一步动作 |
| 语义可理解 | 尽量让业务人员看到业务对象、字段说明和映射关系，而不是只看到技术字段 |
| Agent 有边界 | Agent 入口必须展示权限范围、数据质量和来源限制 |

## 3. 全局导航结构

### 3.1 页面框架

基础平台建议采用左侧主导航 + 顶部全局栏 + 内容区的结构。

```text
┌────────────────────────────────────────────────────────────┐
│ 顶部栏：平台名称 / 全局搜索 / 通知 / 当前用户 / 帮助        │
├───────────────┬────────────────────────────────────────────┤
│ 左侧导航      │ 内容区                                      │
│ - 首页        │ 页面标题 / 面包屑 / 主操作                  │
│ - 数据源      │ 筛选区 / 列表或详情 / 侧栏或抽屉            │
│ - 接入任务    │                                            │
│ - 数据表      │                                            │
│ - 数据目录    │                                            │
│ - 数据质量    │                                            │
│ - 权限审计    │                                            │
│ - 语义模型    │                                            │
│ - 关系图谱    │                                            │
│ - Agent 服务  │                                            │
│ - 系统设置    │                                            │
└───────────────┴────────────────────────────────────────────┘
```

### 3.2 左侧导航

| 一级导航 | 二级页面 |
| --- | --- |
| 首页 | 平台概览、异常提醒 |
| 数据源 | 数据源列表、数据源详情、新增数据源 |
| 接入任务 | 任务列表、任务详情、批次详情、错误清单 |
| 数据表 | 数据表列表、数据表详情、字段详情 |
| 数据目录 | 数据集目录、字段目录、数据集详情 |
| 数据质量 | 质量规则、执行记录、问题清单、数据集质量详情 |
| 权限审计 | 用户、角色、部门、数据权限、敏感字段、审计日志 |
| 语义模型 | 业务对象、对象属性、语义关系、数据映射、对象行动策略 |
| 关系图谱 | 关系边列表、关系查询、关系确认 |
| Agent 数据服务 | Agent 查询入口、工具管理、调用记录、回答审计 |
| 系统设置 | 字典、标签、状态配置、平台参数 |

### 3.3 顶部栏

顶部栏建议包含：

- 平台名称。
- 全局搜索。
- 通知入口。
- 当前用户和角色。
- 帮助入口。
- 当前环境标识，例如测试环境、客户现场环境。

全局搜索优先支持搜索数据源、数据表、数据集、字段、业务对象和接入任务。

## 4. 通用页面结构

### 4.1 列表页结构

```text
页面标题 + 主操作按钮
说明文字或状态摘要
筛选区：关键词 / 类型 / 状态 / 负责人 / 更新时间
数据列表：核心字段 + 状态 + 最近更新 + 操作
批量操作区
分页
```

列表页必须展示：

- 名称。
- 类型。
- 状态。
- 负责人。
- 最近更新时间。
- 关键数量指标。
- 主要操作。

### 4.2 详情页结构

```text
面包屑
标题区：名称 / 状态 / 负责人 / 主操作
摘要区：来源 / 更新时间 / 质量 / 权限 / Agent 可用性
标签页：
- 基本信息
- 字段或配置
- 来源和血缘
- 质量状态
- 权限
- 关联对象
- 日志和审计
```

详情页的重点是让用户快速回答：

- 这是什么。
- 从哪里来。
- 谁负责。
- 现在是否可用。
- 质量是否可信。
- 是否允许 Agent 使用。
- 出问题时应该找谁处理。

### 4.3 新增和编辑页结构

新增和编辑页建议采用分步或分区结构：

```text
基础信息
接入或配置方式
负责人和权限
高级配置
确认和保存
```

对复杂操作，例如 Excel 模板确认、接入任务配置、语义映射，可以采用向导式流程。

## 5. 首页

### 5.1 页面目标

首页用于让平台管理员和数据负责人快速了解基础平台运行情况，发现异常并进入处理页面。

### 5.2 页面结构

```text
首页
├─ 状态总览卡片
│  ├─ 数据源总数 / 正常 / 异常
│  ├─ 接入任务成功 / 失败 / 部分成功
│  ├─ 数据表数量 / 已开放给 Agent 数量
│  ├─ 质量通过 / 警告 / 异常
│  ├─ 语义对象数量 / 映射数量
│  └─ Agent 查询次数 / 失败次数
├─ 待处理事项
│  ├─ 接入失败任务
│  ├─ Excel 解析失败文件
│  ├─ 待确认字段说明
│  ├─ 待确认质量问题
│  └─ 待确认关系边
├─ 最近接入任务
├─ 数据质量概览
├─ Agent 使用概览
└─ 快捷入口
   ├─ 新增数据源
   ├─ 创建接入任务
   ├─ 查看数据目录
   ├─ 执行质量检查
   └─ 进入 Agent 查询
```

### 5.3 关键交互

- 点击状态卡片进入对应列表页，并自动带上筛选条件。
- 点击待处理事项进入问题详情或任务详情。
- 首页不做复杂趋势分析，只做基础状态概览。

### 5.4 页面验收

- 可以看到数据源、接入、质量、语义和 Agent 的基础状态。
- 异常事项可从首页进入处理。
- 首页指标与各模块列表数据一致。

## 6. 数据源模块

### 6.1 数据源列表

```text
数据源
├─ 主操作：新增数据源
├─ 筛选：关键词 / 类型 / 状态 / 负责人 / 标签
└─ 列表列：
   ├─ 数据源名称
   ├─ 类型
   ├─ 接入方式
   ├─ 状态
   ├─ 业务负责人
   ├─ 技术负责人
   ├─ 最近接入时间
   ├─ 关联任务数
   └─ 操作：查看 / 编辑 / 创建接入任务 / 停用
```

### 6.2 新增数据源

新增数据源建议采用分步：

```text
步骤 1：选择数据源类型
步骤 2：填写基础信息
步骤 3：记录接入方式
步骤 4：配置负责人和标签
步骤 5：保存并创建接入任务
```

数据源类型包括 ERP、MES、采购、考勤、数据库、API、Excel、共享盘和其他。

### 6.3 数据源详情

```text
数据源详情
├─ 摘要区：名称 / 类型 / 状态 / 负责人 / 接入方式
├─ 标签页
│  ├─ 基本信息
│  ├─ 接入任务
│  ├─ 产出数据表
│  ├─ 风险说明
│  └─ 操作记录
└─ 主操作：编辑 / 创建任务 / 检测连接 / 停用
```

### 6.4 状态与异常

| 状态 | 页面表现 |
| --- | --- |
| 未接入 | 提示创建接入任务 |
| 接入中 | 展示运行中的任务 |
| 正常 | 展示最近成功时间 |
| 异常 | 展示失败原因和负责人 |
| 停用 | 隐藏高风险操作，仅保留查看 |

## 7. 接入任务模块

### 7.1 接入任务列表

```text
接入任务
├─ 主操作：创建任务
├─ 筛选：数据源 / 类型 / 状态 / 负责人 / 最近执行时间
└─ 列表列：
   ├─ 任务名称
   ├─ 数据源
   ├─ 接入方式
   ├─ 状态
   ├─ 最近执行时间
   ├─ 最近结果
   ├─ 成功数量
   ├─ 失败数量
   └─ 操作：执行 / 查看 / 重试 / 停用
```

### 7.2 创建接入任务

根据接入方式呈现不同配置：

| 接入方式 | 页面重点 |
| --- | --- |
| 数据库同步 | 选择数据表、同步频率、主键或增量字段 |
| API 拉取 | 配置接口地址、参数、认证摘要、拉取频率 |
| 文件导入 | 上传文件、选择格式、确认字段 |
| Excel 导入 | 上传样本、确认模板、配置目录、批量导入 |
| 共享盘扫描 | 配置目录路径、文件类型、增量识别策略 |

### 7.3 任务详情

```text
任务详情
├─ 摘要区：任务名称 / 数据源 / 状态 / 负责人
├─ 当前配置
├─ 最近执行结果
├─ 批次列表
│  ├─ 批次号
│  ├─ 开始时间
│  ├─ 结束时间
│  ├─ 状态
│  ├─ 数据量
│  └─ 错误数
├─ 产出数据表
└─ 日志
```

### 7.4 批次详情与错误清单

批次详情需要展示：

- 批次号。
- 执行时间。
- 文件数或记录数。
- 成功数量。
- 失败数量。
- 跳过数量。
- 错误原因。
- 产出表。

错误清单需要展示：

- 文件名或记录 ID。
- 行号或字段。
- 错误类型。
- 错误说明。
- 建议处理方式。
- 是否可重试。

```text
│  ├─ 起点 ID
│  ├─ 关系名称
│  ├─ 终点对象
│  ├─ 终点 ID
│  ├─ 来源
│  ├─ 生成方式
│  ├─ 可信度
│  └─ 确认状态
├─ 关系查询
└─ 待确认关系
```

### 13.2 关系查询页面

关系查询页面支持输入：

- 对象类型。
- 对象 ID。
- 关系类型。
- 跳数范围。
- 可信度范围。
- 是否仅看已确认关系。

查询结果可以用两种方式展示：

- 路径列表：适合基础阶段。
- 简单关系图：后续增强。

基础阶段优先保证路径列表可读、可追踪。

### 13.3 关系边详情

关系边详情展示：

- from_entity / from_id。
- relation_name。
- to_entity / to_id。
- properties。
- source_dataset。
- generated_by。
- confidence。
- confirmed_by。
- valid_from / valid_to。
- 关联数据证据。

### 13.4 关系确认

待确认关系页面支持：

- 查看关系来源。
- 查看生成方式。
- 查看证据字段。
- 确认关系。
- 拒绝关系。
- 标记为信息不足。

AI 生成的关系默认不应直接进入高可信事实关系。

## 14. Agent 数据服务模块

### 14.1 Agent 查询入口

```text
Agent 数据服务
├─ 查询输入区
│  ├─ 问题输入
│  ├─ 当前用户权限范围提示
│  └─ 可用数据范围提示
├─ 回答区
│  ├─ 直接回答
│  ├─ 数据来源
│  ├─ 更新时间
│  ├─ 质量状态
│  ├─ 使用工具
│  └─ 不确定性说明
├─ 引用和证据区
└─ 调用日志入口
```

基础阶段 Agent 入口应明确提示：

- Agent 只能查询已授权数据。
- Agent 回答依赖平台数据质量。
- Agent 不会自动写回业务系统。

### 14.2 Agent 工具管理

工具列表包括：

- 工具名称。
- 工具类型。
- 关联数据集。
- 关联语义对象。
- 权限要求。
- 风险等级。
- 状态。

基础阶段工具类型包括数据目录检索、权限检查、受控数据查询、质量状态查询、语义检索、图谱查询和审计记录。

### 14.3 Agent 调用记录

调用记录列表包括：

- 时间。
- 用户。
- 问题。
- 使用工具。
- 使用数据集。
- 权限检查结果。
- 回答状态。
- 失败原因。

调用详情需要展示完整工具调用链和回答依据。

## 15. 系统设置模块

系统设置基础页面包括：

- 数据源类型字典。
- 接入方式字典。
- 数据质量状态字典。
- 敏感字段类型。
- 业务标签。
- Agent 行动等级。
- 平台参数。

系统设置不应成为复杂配置中心，基础阶段只保留影响数据治理和页面展示的必要字典。

## 16. 核心流程页面串联

### 16.1 数据接入到 Agent 可用

```text
数据源列表
→ 新增数据源
→ 创建接入任务
→ 任务详情
→ 批次详情
→ 数据表详情
→ 字段说明维护
→ 数据目录详情
→ 权限配置
→ Agent 可用标识
→ Agent 查询入口
```

每一步都需要有返回上一层对象的入口，例如从数据表详情返回接入任务，从 Agent 回答返回使用的数据集。

### 16.2 Excel 批量导入

```text
数据源列表
→ 新增 Excel 数据源
→ 创建 Excel 接入任务
→ 上传模板样本
→ 确认字段解析
→ 配置目录
→ 执行批量导入
→ 查看批次详情
→ 查看错误清单
→ 查看产出数据表
```

### 16.3 语义映射到图谱

```text
数据表详情
→ 选择字段
→ 创建或选择业务对象
→ 创建对象属性
→ 建立 DataMapping
→ 定义 SemanticRelation
→ 生成 BusinessGraphEdge
→ 查看关系边列表
→ Agent 查询关系路径
```

### 16.4 数据质量到 Agent 提示

```text
数据表详情
→ 执行质量检查
→ 查看问题清单
→ 更新质量状态
→ 数据目录展示质量状态
→ Agent 查询读取质量状态
→ Agent 回答说明数据限制
```

## 17. 空状态与异常状态

### 17.1 空状态

| 页面 | 空状态文案 | 主操作 |
| --- | --- | --- |
| 数据源列表 | 当前还没有数据源 | 新增数据源 |
| 接入任务列表 | 当前还没有接入任务 | 创建接入任务 |
| 数据表列表 | 当前还没有接入数据 | 去执行接入任务 |
| 数据目录 | 当前还没有可用数据集 | 查看数据表 |
| 质量规则 | 当前还没有质量规则 | 创建规则 |
| 语义对象 | 当前还没有业务对象 | 创建业务对象 |
| 关系图谱 | 当前还没有关系边 | 配置关系映射 |
| Agent 调用记录 | 当前还没有 Agent 调用记录 | 进入查询入口 |

### 17.2 异常状态

| 异常 | 页面处理 |
| --- | --- |
| 数据源不可连接 | 展示失败原因、检测时间、负责人和重试入口 |
| 接入任务失败 | 展示失败批次、日志、错误清单和重试入口 |
| Excel 解析失败 | 展示文件名、行列位置、错误字段和修正建议 |
| 数据质量异常 | 展示问题字段、样例、影响范围和处理状态 |
| 权限不足 | 展示无权限原因和数据负责人 |
| 语义映射冲突 | 展示冲突字段、冲突对象和待确认入口 |
| 关系边低可信 | 展示生成方式、证据和人工确认入口 |
| Agent 查询失败 | 展示失败原因、使用工具和可联系负责人 |

### 17.3 部分成功状态

接入任务、批量导入、质量检查都可能部分成功。页面必须展示：

- 成功数量。
- 失败数量。
- 跳过数量。
- 已入库数据。
- 未入库数据。
- 是否影响 Agent 使用。
- 下一步处理建议。

## 18. 页面级验收标准

### 18.1 总体验收

页面原型需要满足：

- 左侧导航覆盖基础平台所有一级模块。
- 首页能展示基础平台状态和异常入口。
- 数据源到 Agent 可用的核心流程可以在页面之间串联。
- Excel 批量导入流程有样本确认、批量执行和错误清单。
- 数据表、数据目录、数据质量和权限页面能支撑基础治理。
- 语义模型页面能支撑对象、属性、关系、映射和行动策略维护。
- 关系图谱页面能支撑 BusinessGraphEdge 查询和确认。
- Agent 数据服务页面能展示回答、来源、质量、权限和审计。
- 空状态、异常状态和部分成功状态都有明确处理方式。

### 18.2 进入视觉原型的条件

进入视觉原型前，需要确认：

- 一级导航和二级页面结构已确认。
- 核心流程页面串联已确认。
- 每个模块的列表页和详情页范围已确认。
- Agent 查询入口和回答结构已确认。
- 基础阶段不做事项已确认。

### 18.3 进入研发拆解的条件

进入研发拆解前，需要补充或确认：

- 各页面字段清单。
- 表单校验规则。
- 权限矩阵。
- 状态流转。
- 关键操作审计要求。
- Agent 工具调用边界。
- 页面验收用例。
