## 二、Agent能力全景矩阵

### 2.1 能力等级速查

| 等级 | 能力 | 含义 | 举例 |
|------|------|------|------|
| **L0 查** | 查询/检索 | 你问数据，它回答数据 | "物料编码M-001的库存是多少？" |
| **L1 析** | 分析/解释 | 你问趋势，它分析原因 | "为什么M-001库存一直在降？" |
| **L2 策** | 建议/预警 | 你不需要问，它主动说 | "⚠️ M-001 安全库存告急，建议补500件" |
| **L3 协** | 协同/执行 | 它帮你发起流程 | "需要我帮PO-001发起催货通知吗？" |
| **L4 写** | 写入/回写 | ❌ 默认禁止，需审批 | — |

### 2.2 按业务域 × 能力等级

| 业务域 | L0 查 | L1 析 | L2 策 | 场景数 |
|--------|-------|-------|-------|--------|
| 🏭 生产制造 | 工单/工序/投料 | 延迟归因/报废率 | 欠料预警/排程建议/BOM核对 | 8 |
| 📦 库存管理 | 库存/批次/条码 | 周转率/呆滞分析 | 安全库存/断料/呆滞预警 | 7 |
| 🛒 采购供应 | 订单/在途/价格 | 交期延迟/价格异常 | 催货建议/供应商评级 | 6 |
| 💰 销售分销 | 订单/发货/应收 | 交付率/利润率 | 交付风险/信用预警 | 5 |
| ✅ 质量管理 | 检验结果/不良品 | 合格率趋势/原因分布 | 连续不良/检验超时预警 | 5 |
| 📊 财务管控 | 应收/应付/成本 | 逾期分析/成本异常 | 逾期提醒/信用预警 | 5 |
| 🔗 跨系统 | 跨系统一致性 | 工单/采购/物料对齐 | — | 3 |
| **合计** | **16** | **16** | **16** | **39场景** |

## 三、场景详情：你会怎么用 Agent

> 每个场景按「问题 → Agent做什么 → 需要什么数据 → 数据到位了吗」四段式呈现  
> 状态标记：✅ 数据就绪  🔄 部分就绪  ⏳ 需增补数据

### 🏭 生产制造域（8场景）

#### 场景1｜工单进度查询

```
👤 用户问    MO-088 现在做到哪了？
🤖 Agent    查询工单状态+工序进度+完工率，回复"已确认，工序3/5，完工率62%"
📊 需要数据  ERP MO_CHANGE(状态/计划时间) + MES filterWorkorder(实际进度/completedQty)
📋 数据状态  ✅ ERP 生产制造域已梳理 | ✅ MES接口已登记
```

#### 场景2｜工单延迟归因

```
👤 用户问    哪些工单延迟了？什么原因？
🤖 Agent    扫描所有未完工工单，对逾期工单分析延迟天数，追溯根因（欠料/设备/等待）
📊 需要数据  ERP MO_CHANGE + MO_D(投料) + ITEM_WAREHOUSE(库存) + PURCHASE_ORDER(在途)
📋 数据状态  ✅ 全部已梳理
```

#### 场景3｜工单欠料预警 ⚡ 主动推送

```
👤 触发条件  工单已开工 AND 投料物料库存不足（BR15）
🤖 Agent    主动推送"🔴 MO-088 欠料：M-001缺200件，M-002缺50件。关联采购单PO-001在途→预计3天后到"
📊 需要数据  ERP MO_D(需领量) + ITEM_WAREHOUSE(库存) + PURCHASE_ORDER(在途) + MO_MATERIAL
📋 数据状态  ✅ 全部已梳理 | 🔄 精确交期预测需MRP数据
```

#### 场景4｜报废率分析

```
👤 用户问    上个月报废率为什么超标了？
🤖 Agent    计算报废率=报废量/投产量的趋势，关联原因（来料不良/加工不良/模具异常）
📊 需要数据  ERP MO_PRODUCT(SCRAP_QTY) + MES selectWorkorderReportDataByTime(badQty/badQtyManufacturing/badQtyIncoming)
📋 数据状态  ✅ ERP侧已梳理 | ✅ MES报工接口已登记
```

#### 场景5｜排程冲突预警 ⚡ 主动推送

```
👤 触发条件  同一工作中心同时排了2个工单（BR21:母子工单联动 + 资源冲突）
🤖 Agent    主动推送"⚠️ WorkCenter-03 本周排程冲突：MO-088和MO-089同时占用，建议调整"
📊 需要数据  MES selectProductionBackParamsByTime(排产结果) + ERP MO_CHANGE(计划时间) + WORK_CENTER
📋 数据状态  ✅ ERP已梳理 | ✅ MES排产接口已登记 | 🔄 MRP级排程优化需计划模块数据
```

#### 场景6｜BOM版本核对

```
👤 用户问    MO-088 用的BOM是最新版吗？
🤖 Agent    对比工单BOM版本号 vs 当前生效版本，不一致则提示差异
📊 需要数据  ERP MO_MATERIAL(BOM_ID) + BOM(生效日期/版本)
📋 数据状态  ✅ 全部已梳理
```

#### 场景7｜工单投料分析

```
👤 用户问    MO-088 领料进度怎么样？
🤖 Agent    对比需领量vs已领量，列出欠料物料清单，关联采购在途
📊 需要数据  ERP MO_D(REQUIRED_QTY/ISSUED_QTY) + MES selectTaskUseMaterial(useQty)
📋 数据状态  ✅ ERP已梳理 | ✅ MES投料接口已登记
```

#### 场景8｜母子工单联动分析

```
👤 用户问    母工单延迟了，子工单怎么办？
🤖 Agent    从母工单PARA_MO_ID下钻所有子工单，自动标记级联影响
📊 需要数据  ERP MO_CHANGE(PARA_MO_ID/UP_MO_ID) 自引用FK
📋 数据状态  ✅ ERP自引用关系已梳理
```
