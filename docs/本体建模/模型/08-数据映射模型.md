# 08 — 数据映射模型

> 定义语义对象、属性、关系、指标如何映射到来源系统。数据映射模型是语义层与真实数据资产之间的桥梁，让 Agent 能够从语义对象定位到实际数据。
>
> 数据来源：`docs/数据梳理/跨系统字段映射表.md` — ERP/MES/SRM/TITA 四系统精确映射（表.列 + 接口路径 + 响应字段路径）

## 1. 映射类型

| 映射类型 | 编码前缀 | 作用 |
| --- | --- | --- |
| 对象映射 | `MPOBJ` | 语义对象 → 来源系统对象（表 / 接口资源） |
| 属性映射 | `MPATTR` | 标准属性 → 来源字段（表.列 / 响应字段） |
| 关系映射 | `MPREL` | 语义关系 → 数据关联机制（FK / 中间表 / 规则推导） |
| 指标映射 | `MPMT` | 指标口径 → 依赖对象、字段、过滤条件、聚合逻辑 |

## 2. 四系统映射总览

| 系统 | 类型 | 规模 | 标识字段特征 |
| --- | --- | --- | --- |
| ERP (E10) | 关系数据库 | 26模块374表 | GUID FK (`XXX_ID`)；单据号 (`DOC_NO`)；状态码 (`STATUS`)；自定义字段 (`UDF001`~`UDF054`) |
| MES (LightMES) | REST API | 6类核心接口 | int64 ID (`woid`)；`workorderNo`；`status`(1-9枚举)；`customFields[].fieldCode`/`fieldValue` |
| SRM (携客云) | REST API | 9类接口 | `innerXxxCode`(内部编码)；`xxxStatus`(int枚举)；`platformExtendN01`~`N12` |
| TITA (CRM) | REST API | 客户API | `Name`(名称即标识)；`Status`(string)；`CreatedBy`/`CreatedTime` |

## 3. 实际映射示例（来自跨系统字段映射表）

### 3.1 物料：Material / 物料 (BO-MATERIAL)

**对象映射**：

```yaml
mapping_code: MPOBJ_BO_MATERIAL
mapping_type: MPOBJ
semantic_target: BO-MATERIAL
source_systems:
  - system: ERP(E10)
    database: E10_DB
    module: ITEM
    tables: 51张子表
    primary_table: ITEM
    primary_key: ITEM_ID (GUID)
  - system: MES(LightMES)
    api: POST /lightmesapi/open/materialController/filterMaterial
    key_field: materialNo
  - system: SRM(携客云)
    api: POST /compProd/pvRelation/queryList.json
    key_field: prodCode
  - system: TITA
    api: GET /crm/api/{tenantId}/customer/search
    key_field: Crm.Product.Name (间接关联)
trust_level: 高
status: 启用
```

**属性映射 — 四系统对照**：

| 语义属性 | ERP 来源 | MES 来源 | SRM 来源 |
| --- | --- | --- | --- |
| 物料编码 | `ITEM.ITEM_CODE` | `materialNo` | `prodCode` |
| 物料名称 | `ITEM.ITEM_NAME` | `materialDesc` | `prodName` |
| 规格 | `ITEM.ITEM_SPECIFICATION` | `materialSpec` | `prodScale` |
| 单位 | `ITEM.UNIT` | `unit` | — |

### 3.2 制造工单：ProductionOrder (BO-PRODUCTION-ORDER)

**对象映射 — ERP vs MES 双视角**：

```yaml
mapping_code: MPOBJ_BO_PRODUCTION_ORDER
mapping_type: MPOBJ
semantic_target: BO-PRODUCTION-ORDER
source_systems:
  - system: ERP(E10)
    module: MO
    tables: 41张子表
    primary_table: MO_CHANGE  # 工单变更单(主表)
    primary_key: MO_ID (GUID)
    key_tables:
      - MO_CHANGE: 主表(状态/计划/开工完工日期)
      - MO_PRODUCT: 产出信息(COMPLETED_QTY/SCRAP_QTY)
      - MO_D: 材料明细(ITEM_ID/REQUIRED_QTY/ISSUED_QTY)
      - MO_ROUTING: 工艺信息(ITEM_ROUTING_ID)
      - MO_RECEIPT: 入库信息
      - MO_INSPECTION: 检验信息
  - system: MES(LightMES)
    apis:
      - POST /lightmesapi/open/workorderController/filterWorkorder  # 核心查询
      - POST /lightmesapi/open/workorderController/selectTaskUseMaterial  # 投料明细
      - POST /lightmesapi/open/workorderController/selectWorkorderProcedure  # 工序
      - POST /lightmesapi/open/workorderController/selectProductionBackParamsByTime  # 排产
      - POST /lightmesapi/open/workorderController/selectWorkorderReportDataByTime  # 报工
trust_level: 高
status: 启用
```

**ERP ↔ MES 关键字段对接规则**：

```yaml
cross_system_matching:
  - rule: "ERP MO.DOC_NO = MES workorderNo"
    trust: high
  - rule: "ERP ITEM.ITEM_CODE = MES materialNo"
    trust: high
  - note: "MES特有字段(customerNo/salesorderNo/orderMaster/urgentLevel/goodQty/badQty)在ERP侧无直接对应"
  - note: "ERP特有字段(PARA_MO_ID/UP_MO_ID/BOM_VERSION_TIMES)在MES侧无直接对应"
```

### 3.3 采购订单：PurchaseOrder (BO-PURCHASE-ORDER)

**属性映射 — ERP ↔ SRM 对照**：

```yaml
mapping_code: MPATTR_BO_PURCHASE_ORDER
mapping_type: MPATTR
semantic_target: BO-PURCHASE-ORDER
field_mappings:
  - semantic: po_doc_no
    erp: PURCHASE_ORDER.DOC_NO
    srm: purchaseOrderNo
  - semantic: supplier_id
    erp: PURCHASE_ORDER.SUPPLIER_ID (GUID FK)
    srm: innerVendorCode (String)
  - semantic: status
    erp: PURCHASE_ORDER.STATUS
    srm: closeStatus
  - semantic: amount_untaxed
    erp: PURCHASE_ORDER.AMOUNT_UNINCLUDE_TAX_OC
    srm: amountUnincludeTax
  - semantic: tax_amount
    erp: PURCHASE_ORDER.TAX_OC
    srm: taxAmount
  - semantic: plant_id
    erp: PURCHASE_ORDER.PLANT_ID (GUID FK)
    srm: profitCenterCode (String)
  - semantic: buyer
    erp: PURCHASE_ORDER.Owner_Emp (GUID FK)
    srm: buyerName
trust_level: 高
status: 启用
```

## 4. 四系统数据特征对照

| 特征 | ERP (E10) | MES | SRM | TITA |
| --- | --- | --- | --- | --- |
| 主标识类型 | GUID (`XXX_ID`) | int64 (`xxid`) | String (`innerXxxCode`) | string (`xxxId`) |
| 状态字段 | `STATUS`(string码) | `status`(int32枚举) | `status`/`xxxStatus`(int) | `Status`(string) |
| 创建人 | `CreateBy`(GUID) | `createTuid`/`createUserName` | `sendAuditEmpName` | `CreatedBy`(string) |
| 创建时间 | `CreateDate` | `createTime` | — | `CreatedTime` |
| 备注 | `REMARK` | `remark` | `remark`/`description` | `remark` |
| 自定义字段 | `UDF001`~`UDF054`(固定17组) | `customFields[].fieldCode`/`fieldValue` | `platformExtendN01`~`N12` | — |
| FK类型 | GUID | int64 | String | string |

## 5. 优先映射原则

初期不要求把所有字段都语义化。优先映射：

1. **对象主标识**：`ITEM_CODE` / `DOC_NO` / `workorderNo` / `prodCode` / `innerVendorCode`
2. **关键属性**：名称、状态、日期、数量、金额 ≈184个属性中的核心集
3. **核心关系**：15条 high 可信度的 FK 关系
4. **指标依赖字段**：交付日期、库存数量、完工数量、质检结果
5. **Agent 可查询入口**：物料、订单、工单、库存、客户、供应商 6个高频入口
