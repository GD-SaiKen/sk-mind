# 11 — 制造业行业语义模板

> 面向中小型制造企业的预置业务对象、属性、关系、规则和 Agent 工具映射。基于 ERP(E10) 26模块374表 + MES(LightMES) + SRM(携客云) + TITA(CRM) 真实数据。
>
> 该模板不是强制标准，而是帮助企业快速建立第一版语义本体的默认参考。

## 1. 模板层级

```text
平台级元模型（01-10）：定义如何描述对象、属性、关系、规则、指标、行动和映射
制造业行业模板（本文档）：定义制造企业 20个BO + 41条关系 + 36条规则 + 四系统映射
企业扩展模型：基于行业模板扩展客户自己的对象、叫法、规则和系统映射
```

**优先级说明：**
- **P0（制造价值链核心）**：8个对象覆盖订单到交付全链路，首批启用

- **P2（资产与扩展）**：5个对象覆盖资产设备管理和退货扩展，渐进启用

## 3. 核心关系链（6条 Agent 常用路径）

| # | 用户问题 | Agent 遍历关系链 | 涉及关系 |
|---|---------|-----------------|----------|
| Q1 | "订单 SO-001 能按时交付吗？" | SalesOrder → Shipment → Inventory → ProductionOrder → PurchaseOrder | R16, R18, R22, R13, R03 |
| Q2 | "物料 M-001 为什么库存不足？" | Material → BOM → ProductionOrder(消耗) → PurchaseOrder(在途) → Supplier | R08, R12, R03, R01 |
| Q3 | "供应商 S-001 最近表现如何？" | Supplier → PurchaseOrder(交期) → PurchaseReceipt → InspectionOrder → DefectiveRecord | R02, R07, R32, R34 |
| Q4 | "工单 MO-001 的成本是多少？" | ProductionOrder → Material(投料成本) → Cost → GL | R12, R29, R30 |
| Q5 | "客户 C-001 有没有逾期应收？" | Customer → SalesOrder → AR → Payment | R16, R20, R25 |
| Q6 | "这批来料可以直接入库吗？" | PurchaseReceipt → InspectionOrder → Acceptance | R32, R35 |

## 4. 完整 Agent 使用链路（以"SO-001 交付风险"为例）

```text
用户问题：「SO-001 有没有交付风险？」
        ↓
1. 意图识别：分析类问题（风险分析）
        ↓
2. 业务对象识别：
   核心对象 → BO-SALES-ORDER:SO-001
   关联对象 → BO-CUSTOMER, BO-PRODUCT, BO-BOM, BO-MATERIAL,
             BO-INVENTORY, BO-PURCHASE-ORDER, BO-PRODUCTION-ORDER, BO-SHIPMENT
        ↓
3. 关系路径规划（沿业务关系图谱边）：
   Customer ──places──→ SalesOrder
   SalesOrder ──references──→ Material
   Material ──tracked_by──→ Inventory
   Material ──purchased_by──→ PurchaseOrder (在途)
   Product ──produced_by──→ ProductionOrder (在产)
   SalesOrder ──ships_via──→ Shipment
        ↓
4. 数据映射定位：
   BO-SALES-ORDER → ERP SALE_CARD_DOC / MES salesorderNo / SRM saleOrderNo
   BO-INVENTORY → ERP ITEM_WAREHOUSE / MES warehouse
   BO-PURCHASE-ORDER → ERP PURCHASE_ORDER / SRM purchaseOrderNo
   BO-PRODUCTION-ORDER → ERP MO_CHANGE / MES filterWorkorder
        ↓
5. 数据质量与可信检查：
   检查数据更新时间、完整性、跨系统一致性
   ERP vs MES 工单状态对照：ERP STATUS码 vs MES status(1-9)
        ↓
6. 工具调用（按 L0-L2 风险等级）：
   L0: get_order_detail(SO-001)
   L1: check_material_availability(SO-001关联物料)
   L1: get_purchase_status(SO-001关联物料的采购在途)
   L1: get_production_progress(SO-001关联产品的生产订单)
        ↓
7. 规则判断：
   BR15 MO_MATERIAL_SHORTAGE → 检查工单欠料
   BR09 PO_DELIVERY_DELAY → 检查采购延迟
   BR16 MO_SCHEDULE_DELAY → 检查生产进度
   BR01 INV_SAFETY_STOCK_ALERT → 检查安全库存
        ↓
8. 回答生成（含依据/来源/不确定性/建议）：
   【结论】订单 SO-001 存在中等交付风险（L2）
   【关键依据】关键物料 MAT-001 库存50 < 安全库存100；PO-001 预计到货6/20
   【数据来源】库存-MES(6/13 08:00)；采购-ERP(6/13 06:00)；生产-MES(6/13 07:30)
   【关系路径】SO-001→订单行→PROD-001→BOM→MAT-001→库存/采购/生产
   【不确定性】采购到货为供应商承诺；生产排程可能调整
   【建议】联系供应商确认PO-001；考虑替代物料或调整排程
```

## 6. 企业扩展指南

基于此模板落地时，建议按以下步骤扩展：

1. **确认对象覆盖**：对照企业实际 ERP/MES/SRM 系统，确认哪些 BO 适用，并增加企业特有对象
2. **补充属性和叫法**：将标准属性映射到企业实际使用的字段名和叫法（不同 ERP 版本字段名可能不同）
3. **建立关系路径**：根据企业系统实际外键和关联关系，补充或修正关系定义
4. **配置数据映射**：将语义 BO/属性/关系映射到 ERP 表.列、MES 接口、SRM 接口
5. **定义指标口径**：确认指标计算公式与企业实际口径一致
6. **设置行动权限**：根据企业组织架构和权限策略，调整对象行动策略
7. **渐进启用**：P0（物料、订单、工单、库存、客户、供应商、BOM、产品）→ P1（财务+质量）→ P2（资产+扩展）
