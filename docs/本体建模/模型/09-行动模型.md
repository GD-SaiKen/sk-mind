# 09 — 行动模型

> 定义 Agent 能做什么、不能做什么、在什么条件下做、是否需要人确认。与对象行动模型（10）配套：行动模型定义 Agent 具备哪些**类型**的动作能力，对象行动模型定义这些能力能否作用在某类**对象**上。
>
> 数据来源：`docs/本体建模/业务对象全景图.md` 各BO的行为定义

## 1. 行动分类与风险等级

| 行动分类 | 编码前缀 | 风险等级 | 作用 |
| --- | --- | --- | --- |
| 查询行动 | ACTQRY | L0 只读 | 读取、检索、筛选、汇总、明细展开 |
| 分析行动 | ACTANL | L1 分析 | 计算、对比、归因、趋势分析 |
| 解释行动 | ACTEXP | L1 分析 | 解释指标、异常、数据来源和不确定性 |
| 建议行动 | ACTSUG | L2 建议 | 生成建议、备选方案、影响分析 |
| 提醒行动 | ACTALT | L2 建议 | 预警、通知、待办 |
| 协同行动 | ACTCOL | L3 协同 | 生成任务、发起确认、提交审批 |
| 写入行动 | ACTWRT | L4 写入 | 新增、修改、关闭、回写业务系统 |
| 治理行动 | ACTGOV | L1-L3 | 标注、纠错、申请权限、质量工单 |

## 2. 风险等级分层策略

| 等级 | 名称 | 能力范围 | 默认策略 |
| --- | --- | --- | --- |
| L0 | 只读 | 查询、检索、查看 | 默认允许，但受权限控制 |
| L1 | 分析 | 计算、对比、解释 | 默认允许，但需引用指标口径和数据来源 |
| L2 | 建议 | 生成建议、预警、待办草稿 | 默认允许，但必须标明建议依据和不确定性 |
| L3 | 协同 | 发起确认、提交审批、创建任务 | 需要用户确认 |
| L4 | 写入 | 修改数据、回写系统、触发业务动作 | 默认禁止自动执行，需审批和审计 |

## 3. 制造业预置行动（按BO分类）

### 3.1 Material / 物料 (BO-MATERIAL)

| 等级 | 动作 | 说明 | 风险 |
| --- | --- | --- | --- |
| L0 | 查询物料详情 | 查看编码、名称、规格、单位、分类 | 低 |
| L0 | 查询物料库存 | 查看当前库存量、可用量、在途量 | 低 |
| L0 | 查询物料供应商 | 查看哪些供应商可供应此物料 | 低 |
| L1 | 分析库存趋势 | 过去N天库存变化、消耗速率 | 低 |
| L1 | 分析采购价格 | 不同供应商报价对比、价格趋势 | 低 |
| L2 | 建议补货 | 基于安全库存+消耗速率生成补货建议 | 中 |
| L2 | 预警缺料 | 库存低于安全库存时触发预警 | 中 |
| — | ❌ 修改物料主数据 | 需采购/工程部门审批 | 禁止 |

### 3.2 PurchaseOrder / 采购订单 (BO-PURCHASE-ORDER)

| 等级 | 动作 | 说明 |
| --- | --- | --- |
| L0 | 查询采购订单 | 查看单据详情、行明细 |
| L0 | 查询采购在途 | 关联到货/收货状态 |
| L1 | 分析交期延迟 | 超期未到货统计 |
| L1 | 分析价格异常 | 单价偏离均值检测 |
| L2 | 建议催货 | 基于交期延迟+工单欠料 |
| L2 | 预警交期风险 | 临近交期未完全收货 |
| — | ❌ 创建/修改采购订单 | 需采购员操作 |

### 3.3 ProductionOrder / 制造工单 (BO-PRODUCTION-ORDER)

| 等级 | 动作 | 说明 |
| --- | --- | --- |
| L0 | 查询工单详情 | 基本信息、工序进度、投料明细 |
| L0 | 查询工单进度 | 完工数量、剩余数量、完工率 |
| L1 | 分析工单延迟 | 延迟天数、延迟原因关联 |
| L1 | 分析投料情况 | 已领量 vs 需领量、欠料清单 |
| L1 | 分析报废率 | 报废率 vs 预设阈值 |
| L2 | 预警欠料 | 欠料影响工单进度时触发 |
| L2 | 建议调整排程 | 基于延迟+资源冲突 |
| L2 | 预警BOM版本不一致 | 工单BOM ≠ 当前生效版本 |
| — | ❌ 关闭/修改工单 | 需生产主管审批 |

### 3.4 SalesOrder / 销售订单 (BO-SALES-ORDER)

| 等级 | 动作 |
| --- | --- |
| L0 | 查询销售订单详情 |
| L1 | 分析订单交付率 |
| L1 | 分析订单利润率 |
| L2 | 预警交付风险（关联工单+库存） |
| L2 | 建议客户信用检查 |
| — | ❌ 创建/取消销售订单 |

### 3.5 Inventory / 库存 (BO-INVENTORY)

| 等级 | 动作 |
| --- | --- |
| L0 | 查询库存量 |
| L1 | 分析库存周转率 |
| L1 | 分析呆滞库存（90天无消耗） |
| L2 | 预警安全库存不足 |
| L2 | 预警呆滞库存 |
| L2 | 预警负库存异常 |
| — | ❌ 直接修改库存数量（需通过出入库单据） |

### 3.6 财务对象行动限制

| 对象 | L0-L1允许 | L2允许 | L4禁止 |
| --- | --- | --- | --- |
| AR (BO-AR) | 查询应收、分析逾期 | 逾期提醒 | 禁止修改余额 |
| AP (BO-AP) | 查询应付、分析到期 | 到期提醒 | 禁止修改余额 |
| GL (BO-GL) | ⚠️ 仅聚合查询 | — | 禁止查看明细行 |
| Cost (BO-COST) | 分析趋势和异常 | — | 禁止修改成本数据 |

### 3.7 Inspection / 检验管理 (BO-INSPECTION)

| 等级 | 动作 |
| --- | --- |
| L0 | 查询检验结果 |
| L1 | 分析合格率趋势 |
| L1 | 分析不良原因分布 |
| L2 | 预警检验超时 |
| L2 | 预警连续不良趋势 |
| — | ❌ 修改检验结果 |

## 4. 工具映射（制造业预置）

| 工具名称 | 作用 | 关联对象 | 风险等级 |
| --- | --- | --- | --- |
| `get_order_detail` | 查询订单详情 | BO-SALES-ORDER, BO-CUSTOMER | L0 |
| `get_order_materials` | 查询订单相关物料需求 | BO-SALES-ORDER, BO-PRODUCT, BO-BOM, BO-MATERIAL | L1 |
| `check_material_availability` | 检查物料库存和可用性 | BO-MATERIAL, BO-INVENTORY, BO-PURCHASE-ORDER | L1 |
| `get_purchase_status` | 查询采购订单和在途状态 | BO-PURCHASE-ORDER, BO-SUPPLIER, BO-MATERIAL | L0 |
| `get_production_progress` | 查询生产订单进度 | BO-PRODUCTION-ORDER, BO-PRODUCT, BO-MATERIAL | L0 |
| `get_inventory_status` | 查询库存状态 | BO-INVENTORY, BO-MATERIAL | L0 |
| `analyze_delivery_risk` | 分析交付风险 | BO-SALES-ORDER, BO-INVENTORY, BO-PURCHASE-ORDER, BO-PRODUCTION-ORDER, BO-SHIPMENT | L2 |
| `analyze_metric_trend` | 指标趋势分析 | 指标对象 | L1 |
| `suggest_replenishment` | 补货建议 | BO-MATERIAL, BO-INVENTORY | L2 |
| `create_correction_ticket` | 发起数据纠错工单 | 全部 | L3 |
| `request_confirmation` | 请求业务确认 | 全部 | L3 |

> 工具映射不等于工具实现。先定义「某类问题可调用哪些工具、依赖哪些对象、风险等级、是否需要确认」，后续实施时映射到查询接口、指标服务、规则服务或外部系统 API。
