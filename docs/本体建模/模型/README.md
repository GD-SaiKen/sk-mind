> 版本：v1.0
> 日期：2026-07-04
> 来源：基于《AI 原生企业数据平台产品方案》第 5 章 + 本体建模三份详细产出文档 + 跨系统字段映射表
>
> 数据基础：ERP(E10) 26核心模块374表 + MES(LightMES) 6类接口 + SRM(携客云) 9类接口 + TITA(CRM) 客户API

## 概述

本目录包含平台轻量语义本体层的全部子模型定义。平台采用轻量语义本体思想，不要求第一阶段使用完整 RDF、OWL、三元组存储或复杂推理机。

语义本体层的核心目的，是让 Agent 知道：

- 企业里有哪些业务对象
- 这些对象有哪些属性
- 对象类型之间如何关联
- 具体对象实例之间形成了哪些业务关系路径
- 每类对象允许执行哪些查询、分析、解释、建议、协同或写入动作
- 指标如何计算
- 规则如何判断
- 真实数据在哪里
- 哪些数据可以使用
- 哪些行动可以执行
- 哪些行动必须经过人工确认

## 模型关系

```
对象模型 ──┬── 属性模型（对象有哪些属性）
           ├── 关系模型（41条SemanticRelation：对象类型间连接规则）
           ├── 事件状态模型（生命周期 + 状态流转 + ERP/MES状态码对照）
           ├── 规则模型（36条BusinessRule：校验/判断/预警/权限/Agent边界）
           ├── 指标模型（5大核心制造指标：公式/维度/口径）
           └── 数据映射模型（四系统≈88条映射：表.列 + 接口路径）

行动模型 ──── 对象行动模型（L0-L4分级 × 20个BO策略矩阵）

关系模型 ──── 业务关系图谱模型（45条实例边 + 可信度 + Agent查询路径）

数据映射模型 ── 连接语义层与 ERP/MES/SRM/TITA 真实数据资产
```

其中：

- `关系模型`（SemanticRelation）与 `业务关系图谱模型`（BusinessGraphEdge）配套：前者定义41条关系**类型**，后者记录45条关系**实例**边的生成逻辑
- `行动模型`（AgentAction）与 `对象行动模型`（ObjectActionPolicy）配套：前者定义 Agent 8类**能力**，后者定义每类BO的策略矩阵
- `数据映射模型` 是语义层与四系统（ERP 374表 + MES 6接口 + SRM 9接口 + TITA API）的桥梁

## 轻量化边界

- 不要求第一阶段使用完整 RDF、OWL 或三元组存储
- 不要求业务人员维护复杂知识图谱
- 不要求所有数据字段都进入语义层
- 不要求所有关系都自动推理
- 不替代数据仓库、指标系统、权限系统或工作流系统

语义本体层首先是语义控制面和 Agent 使用数据的导航系统，后续可根据成熟度逐步增强。
