# 10 — 对象行动模型

> 定义每类业务对象可被执行哪些动作、动作风险等级、权限要求、确认要求和审计要求。与行动模型（09）配套。
>
> 数据来源：`docs/本体建模/业务对象全景图.md` 各BO行为定义 + Agent行动边界规则(BR32-BR36)

## 1. 按对象分类的默认策略矩阵

| 对象分类 | 允许行动 | 默认限制 |
| --- | --- | --- |
| 主数据对象 | 查询、解释、质量校验、发起纠错 | 修改主数据需人工确认或审批 |
| 交易对象 | 查询、分析、状态解释、风险提醒 | 关闭、撤销、回写状态默认禁止自动执行 |
| 资源对象 | 查询、占用分析、利用率分析、预警建议 | 自动调度或分配资源需确认 |
| 过程对象 | 查询、进度分析、异常解释 | 创建、关闭、重开工单需确认或审批 |
| 财务对象 | 查询、分析、到期提醒 | 禁止修改余额/成本数据；GL仅聚合查询 |
| 事件对象 | 查询、影响分析、归因解释 | 创建事件需确认（自动监测生成除外） |
| 指标对象 | 查询、计算、对比、解释、归因 | 修改指标口径需治理流程 |
| 规则对象 | 查询、解释、模拟命中、建议调整 | 启停规则或修改阈值需审批 |
| 数据对象 | 查询元数据、查看血缘、质量分析、发起治理工单 | 直接删除、覆盖或批量修改数据默认禁止 |

## 2. 核心对象详细策略

### 2.1 Material / 物料 (BO-MATERIAL)

| 字段 | 值 |
| --- | --- |
| policy_code | `POLICY_BO_MATERIAL` |
| object_type | BO-MATERIAL |
| object_category | 主数据 |
| allowed_actions | 查询详情、查询库存、查询供应商、分析库存趋势、分析采购价格 |
| forbidden_actions | 修改物料主数据 |
| conditional_actions | 建议补货（需安全库存规则触发）、预警缺料（需低于安全库存） |
| default_risk_level | L0 |
| permission_roles | 采购、工程、仓储、生产计划 |
| confirmation_required | false（查询/分析）；true（修改/纠错） |
| audit_required | true |

## 3. 策略执行检查链

Agent 调用任何工具前，必须经过以下检查链：

```text
用户请求
  ↓
1. 识别目标业务对象类型 (e.g. BO-PRODUCTION-ORDER)
  ↓
2. 查询对象行动策略 (ObjectActionPolicy)
  ↓
3. 检查请求的行动是否在 allowed_actions 中
   → 若在 forbidden_actions → 拒绝并提示用户
   → 若在 conditional_actions → 检查前置条件
  ↓
4. 检查 Agent 行动边界规则 (BR32-BR36):
   - BR32: L4写入 → 阻断，要求人工审批
   - BR33: 数据质量异常 → 附带质量警告
   - BR34: 高敏字段 → 脱敏展示
   - BR35: 跨系统写入 → 2级审批
   - BR36: 低可信度 → 标注"建议人工核实"
  ↓
5. 检查权限 (permission_roles)
  ↓
6. 检查是否需要人工确认 (confirmation_required)
  ↓
7. 执行工具调用
  ↓
8. 审计记录 (audit_required)
```

这个检查链确保 Agent 的每次行动都落在对象行动策略和 Agent 边界规则的约束范围内。
