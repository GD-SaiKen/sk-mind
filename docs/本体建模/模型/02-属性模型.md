# 02 — 属性模型

> 定义对象有哪些标准属性。属性模型描述每个业务对象的信息结构，是 Agent 理解数据字段语义的基础。
>
> 数据来源：ERP(E10) 表.列 + MES 接口响应字段 + SRM 接口响应字段 + TITA API 字段

## 1. 属性分类

属性按业务含义分为以下类别：

| 分类 | 代码 | 含义 | 示例 |
| --- | --- | --- | --- |
| 标识属性 | `IDENTIFIER` | 唯一识别对象 | 编码、编号、GUID、外部系统 ID、版本号 |
| 描述属性 | `DESCRIPTIVE` | 描述对象特征 | 名称、规格、型号、分类、备注 |
| 状态属性 | `STATUS` | 记录对象当前状态 | 订单状态、审批状态、生命周期状态 |
| 时间属性 | `TEMPORAL` | 记录时间节点 | 创建时间、交期、计划开始时间、实际完工时间 |
| 度量属性 | `QUANTITATIVE` | 可度量的数值 | 数量、金额、工时、比率、重量 |
| 规则属性 | `RULE` | 承载规则阈值 | 安全库存、最小起订量、提前期、质检规则标记 |
| 关联属性 | `REFERENCE` | 引用其他对象 | FK外键、来源单号、关联对象ID |
| 来源属性 | `SOURCE` | 数据溯源 | 来源系统、来源表、来源接口 |

## 2. 属性定义模板

每个属性按以下模板定义，含跨系统精确来源：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| attribute_code | string | Y | 属性编码，全局唯一 |
| attribute_name | string | Y | 属性中文名称 |
| attribute_name_en | string | N | 属性英文名称 |
| owner_object | string | Y | 所属对象编码，如 `BO-MATERIAL` |
| attribute_category | enum | Y | 属性分类 |
| data_type | enum | Y | 数据类型 |
| is_required | boolean | Y | 是否必填 |
| is_unique | boolean | N | 是否唯一 |
| enum_constraint | string | N | 枚举约束 |
| sensitivity_level | enum | Y | 敏感等级：普通 / 内部 / 敏感 / 高敏 |
| agent_available | boolean | Y | Agent 是否可用 |
| source_erp | string | N | ERP来源：`表.列` |
| source_mes | string | N | MES来源：接口.响应字段 |
| source_srm | string | N | SRM来源：接口.响应字段 |
| source_tita | string | N | TITA来源：API.响应字段 |
