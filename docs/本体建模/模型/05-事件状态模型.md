# 05 — 事件状态模型

> 定义对象什么时候发生变化、状态如何流转、哪些事件会触发规则或行动。基于真实 ERP/MES 状态码和生命周期定义。
>
> 数据来源：20个BO的生命周期定义 + ERP状态码 + MES status枚举

## 1. 事件定义模板

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| event_code | string | Y | 事件编码 |
| event_name | string | Y | 事件名称 |
| event_category | enum | Y | 创建 / 变更 / 状态转移 / 异常 / 外部 |
| applicable_objects | list | Y | 适用对象编码 |
| pre_state | string | Y | 前置状态 |
| target_state | string | Y | 目标状态 |
| trigger_rules | list | N | 触发哪些规则（引用06-规则模型） |
| trigger_actions | list | N | 触发哪些 Agent 行动（引用09-行动模型） |
| source_system | string | N | 事件来源系统 |
| audit_required | boolean | Y | 是否需要审计 |

## 2. 核心对象生命周期与状态码

### 2.1 Material / 物料 (BO-MATERIAL)

```text
新建 → 审核 → 生效 → 停用
```

| 状态码 | 状态名称 | ERP状态 | 说明 |
| --- | --- | --- | --- |
| NEW | 新建 | — | 物料主数据创建 |
| APPROVED | 已审核 | `ApproveStatus` | 通过审核 |
| ACTIVE | 生效 | — | 可被引用使用 |
| INACTIVE | 停用 | — | 不可再被新单据引用 |

### 2.2 Supplier / 供应商 (BO-SUPPLIER)

```text
潜在 → 认证 → 合格 → 冻结 → 淘汰
```

| 状态码 | 状态名称 | ERP状态 | SRM状态 |
| --- | --- | --- | --- |
| POTENTIAL | 潜在 | — | — |
| CERTIFIED | 认证 | — | — |
| QUALIFIED | 合格 | `ApproveStatus`=已审核 | `status`=1(有效) |
| FROZEN | 冻结 | — | `status`=0(无效) |
| ELIMINATED | 淘汰 | — | — |

### 2.3 Customer / 客户 (BO-CUSTOMER)

```text
潜在 → 意向 → 成交 → 活跃 → 休眠 → 流失
```

| 状态码 | 状态名称 | TITA来源 | ERP来源 |
| --- | --- | --- | --- |
| POTENTIAL | 潜在 | `Status` | — |
| INTENT | 意向 | `Stage` | — |
| DEAL | 成交 | `Status` | — |
| ACTIVE | 活跃 | `Status` | `ApproveStatus`=已审核 |
| DORMANT | 休眠 | — | — |
| CHURNED | 流失 | — | — |

### 2.4 BOM / 物料清单 (BO-BOM)

```text
草稿 → 审核 → 生效 → 变更中 → 失效
```

| 状态码 | 状态名称 | ERP字段 |
| --- | --- | --- |
| DRAFT | 草稿 | — |
| APPROVED | 已审核 | — |
| EFFECTIVE | 生效 | `EFFECTIVE_DATE ≤ 当前日期` |
| CHANGING | 变更中 | — |
| EXPIRED | 失效 | `EXPIRY_DATE < 当前日期` |

### 2.5 PurchaseOrder / 采购订单 (BO-PURCHASE-ORDER)

```text
未审核 → 已审核 → 部分收货 → 已收货 → 已结案 → 已关闭
```

| 状态码 | 状态名称 | ERP | SRM |
| --- | --- | --- | --- |
| UNAPPROVED | 未审核 | `STATUS` | — |
| APPROVED | 已审核 | `STATUS` | — |
| PARTIAL_RECEIVED | 部分收货 | `STATUS` | — |
| FULLY_RECEIVED | 已收货 | `STATUS` | — |
| CLOSED | 已结案 | `CLOSE`(结束码) | `closeStatus` |
| CANCELLED | 已关闭 | — | — |

### 2.6 ProductionOrder / 制造工单 (BO-PRODUCTION-ORDER)

ERP vs MES 双系统状态码对照：

| 状态码 | 状态名称 | ERP `STATUS` | MES `status` |
| --- | --- | --- | --- |
| UNSCHEDULED | 未排程 | 状态码 | — |
| SCHEDULED | 已排程 | 状态码 | — |
| UNSTARTED | 未开工 | 状态码 | — |
| IN_PROGRESS | 生产中 | 状态码 | 3(已确认) |
| EXCEPTION | 异常 | — | 2(异常) |
| PENDING_CONFIRM | 待确认 | — | 1(待确认) |
| PARTIAL_CONFIRMED | 部分确认 | — | 5(部分确认) |
| COMPLETED | 已完工 | 状态码 | — |
| CLOSED | 已结案 | 状态码 | 9(已结产) |

MES `status` 枚举：`1`待确认 / `2`异常 / `3`已确认 / `5`部分确认 / `9`已结产

MES `urgentLevel` 枚举：`0`特别加急 / `1`非常紧急 / `2`紧急 / `9`非紧急

### 2.8 Inventory / 库存 (BO-INVENTORY)

```text
正常 → 低于安全库存 → 呆滞 → 冻结 → 报废
```

| 状态码 | 状态名称 | 触发条件 |
| --- | --- | --- |
| NORMAL | 正常 | 可用量 ≥ 安全库存 |
| BELOW_SAFETY | 低于安全库存 | 可用量 < 安全库存 |
| STOCKOUT | 断料 | 可用量 ≤ 0 |
| SLOW_MOVING | 呆滞 | 90天无消耗且库存 > 0 |
| FROZEN | 冻结 | 质检不合格/批次锁定 |
| SCRAPPED | 报废 | 过期/损坏 |

### 2.9 财务对象生命周期

| 对象 | 状态流转 |
| --- | --- |
| AR (BO-AR) | 未收 → 部分收款 → 已收 → 逾期(>30天) → 坏账 |
| AP (BO-AP) | 未付 → 部分付款 → 已付 → 逾期 |
| Asset (BO-ASSET) | 购入 → 使用中 → 折旧中 → 处置 |

### 2.10 Inspection / 检验管理 (BO-INSPECTION)

| 状态码 | 状态名称 | ERP `DECISION` | SRM `verifyResult` |
| --- | --- | --- | --- |
| PENDING | 待检 | — | — |
| INSPECTING | 检验中 | — | — |
| QUALIFIED | 合格 | 合格 | 合格 |
| UNQUALIFIED | 不合格 | 不合格 | 不合格 |
| MRB | MRB处理 | — | — |

### 2.11 Shipment / 发货 (BO-SHIPMENT)

```text
待发货 → 部分发货 → 已发货 → 已签收
```

## 3. 关键事件触发表

基于36条业务规则的事件触发关系：

| 事件 | 触发条件 | 触发规则 | 触发Agent行动 | 严重等级 |
| --- | --- | --- | --- | --- |
| 安全库存预警 | 物料可用量 < 安全库存 | `INV_SAFETY_STOCK_ALERT` (BR01) | L2 补货建议 | ⚠️ 警告 |
| 断料风险 | 安全库存>0 且 可用量≤0 | `INV_STOCKOUT_RISK` (BR02) | L2 紧急缺料通知 | 🔴 严重 |
| 呆滞库存预警 | 库存>最大库存 且 90天无消耗 | `INV_EXCESS_INVENTORY` (BR03) | L2 呆滞处理建议 | ⚡ 提示 |
| 采购交期延迟 | 当前日期>预计到货日 且 未完全收货 | `PO_DELIVERY_DELAY` (BR09) | L2 催货建议 | 🔴 严重 |
| 采购到货临近 | 预计到货日 ≤ 3天 | `PO_DELIVERY_APPROACHING` (BR10) | L2 到货提醒 | ⚡ 提示 |
| 工单欠料 | 工单已开工 且 投料物料库存不足 | `MO_MATERIAL_SHORTAGE` (BR15) | L2 缺料预警 | 🔴 严重 |
| 工单进度延迟 | 当前日期>计划完工日 且 未完工 | `MO_SCHEDULE_DELAY` (BR16) | L1 延迟分析 | 🔴 严重 |
| 工单未按时开工 | 当前日期>计划开工日 且 未开工 | `MO_OVERDUE_START` (BR17) | L1 延迟分析 | ⚠️ 警告 |
| 工单报废率异常 | 报废率 > 预设阈值 | `MO_SCRAP_RATE_HIGH` (BR19) | L2 原因分析建议 | ⚠️ 警告 |
| BOM版本不一致 | 工单BOM版本 ≠ 当前生效版本 | `MO_BOM_VERSION_MISMATCH` (BR20) | L1 版本差异分析 | ⚡ 提示 |
| 来料检验不合格 | 检验结果 = 不合格 | `QC_INCOMING_FAIL` (BR22) | L1 不合格分析 | 🔴 严重 |
| 质量趋势异常 | 连续3批同物料同类缺陷 | `QC_TREND_ABNORMAL` (BR26) | L2 工艺排查建议 | ⚠️ 警告 |
| 应收逾期 | 应收款逾期 > 30天 | `FIN_AR_OVERDUE` (BR27) | L2 催款提示 | 🔴 严重 |
| 客户信用超限 | 已用额度 > 信用额度 | `FIN_CREDIT_LIMIT` (BR29) | L2 暂停新订单建议 | 🔴 严重 |
| 成本异常波动 | 单位成本偏离上月 ±30% | `FIN_COST_ABNORMAL` (BR30) | L1 成本异常分析 | ⚠️ 警告 |
