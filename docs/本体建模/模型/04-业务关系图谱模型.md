# 04 — 业务关系图谱模型

> 定义具体业务对象**实例**之间的关系边。与关系模型（类型层面）配套：关系模型定义41条关系类型，业务关系图谱记录45条边的生成逻辑。
>
> 边生成参考：`docs/数据梳理/ERP/脚本/generate_edges.py` — 45条 BusinessGraphEdge

## 1. 模型定位

| 维度 | 关系模型（SemanticRelation） | 业务关系图谱模型（BusinessGraphEdge） |
| --- | --- | --- |
| 层面 | 类型层面 | 实例层面 |
| 描述内容 | 对象类型之间允许存在什么关系 | 具体对象实例之间实际存在什么关系 |
| 示例 | `CUSTOMER_PLACES_SALES_ORDER` | Customer:CUST-A →(places)→ SalesOrder:SO-001 |
| 使用场景 | 关系类型定义、路径规划 | 具体业务查询、路径追踪、影响分析 |
| 变更频率 | 低（模型设计阶段） | 高（随业务数据同步更新） |
| 数据来源 | 本体建模设计 | 从ERP外键/中间表/接口匹配实时生成 |

## 2. 图谱边定义模板

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| business_graph_edge_id | string | Y | 关系边唯一标识 |
| relation_code | string | Y | 关系类型编码，引用03-关系模型 |
| relation_name | string | Y | 关系名称（动词） |
| from_entity_type | string | Y | 起点对象类型编码 |
| from_entity_id | string | Y | 起点对象实例 ID |
| to_entity_type | string | Y | 终点对象类型编码 |
| to_entity_id | string | Y | 终点对象实例 ID |
| edge_properties | dict | N | 关系属性，如数量、单位、版本 |
| confidence | enum | Y | 关系可信度：high / medium / low |
| generated_by | enum | Y | 生成方式：system / mapping_rule / ai / manual |
| confirmed_by | string | N | 人工确认人，AI生成的关系需确认后提升可信度 |
| source_dataset | string | Y | 来源数据集：ERP表名 / MES接口 / SRM接口 |
| valid_from | datetime | N | 关系生效时间 |
| valid_to | datetime | N | 关系失效时间 |

## 3. 可信度分级策略

| 生成方式 | 默认可信度 | 说明 | Agent处理 |
| --- | --- | --- | --- |
| system | high | 从业务系统外键、关联表直接同步生成，作为事实关系 | 直接使用 |
| mapping_rule | high | 通过明确的数据映射规则推导 | 直接使用 |
| ai | low | AI 推断生成 | 标注「低可信度，建议人工核实」 |
| manual | high | 人工手动创建或确认 | 直接使用 |

## 4. 制造价值链核心关系链（图谱边全景）

源自 ERP `generate_edges.py` 45条边定义。

[[制造价值链核心关系链.mermaid]]

**关系链业务含义：**
- **需求端**：客户下达销售订单 → 订单关联物料/产品 → 发货消耗库存 → 产生应收 → 收款核销 → 过账总账
- **供应端**：供应商供应物料 → 采购订单包含物料 → 请购转采购 → 来料检验 → 检验不合格产生不良品 → 验收判定
- **生产端**：BOM 定义物料组成 → 工单使用 BOM → 工单消耗物料/产出产品 → 完工入库 → 工单执行于工作中心
- **财务端**：成本核算分配至物料 → 过账总账；应收/应付核销入账
- **跨系统**：ERP 工单号 = MES 工单号；ERP 采购单号 = SRM 采购单号；ERP 物料编码 = MES/SRM 物料编码

## 6. Agent 使用业务关系图谱的典型流程

```text
用户问题：「SO-001 有没有交付风险？」
        ↓
1. 识别核心对象：SalesOrder:SO-001
        ↓
2. 查找关系类型（从 03-关系模型）：
   Customer ──places──→ SalesOrder
   SalesOrder ──references──→ Material
   SalesOrder ──ships_via──→ Shipment
   SalesOrder ──generates──→ AR
        ↓
3. 沿图谱边追踪实例（从本模型）：
   SO-001
     →(references)→ MAT-001, MAT-002 (订单行物料)
     →(tracked_by)→ Inventory: MAT-001库存不足
     →(consumed_by)→ ProductionOrder: MO-001 未完工
     →(purchased_by)→ PurchaseOrder: PO-001 在途未到
     →(shipped_by)→ Shipment: 尚未发货
        ↓
4. 汇总信息 → 规则判断 → 风险结论
```

Agent 的查询过程不是"猜表名和字段"，而是：识别核心对象 → 查找可用关系类型 → 沿业务关系图谱边展开相关对象 → 根据数据映射找到真实数据 → 调用工具完成查询、计算或判断。
