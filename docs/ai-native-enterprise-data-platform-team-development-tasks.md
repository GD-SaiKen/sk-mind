# AI 原生企业数据平台团队开发任务拆解

版本：草案 v0.2

日期：2026-06-13

## 文档说明

本文档用于把产品方案、基础建设方案、PRD、技术设计和实施计划拆解为团队可认领的一人周任务。

本次修正的重点是让团队开发任务与《AI 原生企业数据平台落地阶段路线图》一一对应。每个阶段都明确：

- 对应路线图阶段。
- 对应产品建设方案。
- 对应 PRD 或页面规格。
- 对应研发任务。
- 阶段验收边界。

每个任务按 **一人周** 规模设计，即一名成员约 5 个工作日完成，包括需求理解、实现、测试、自测、评审修改和必要文档更新。

任务拆解遵循当前技术方案：

- 前端：Vue 3 + Vite + TypeScript + Element Plus。
- 后端：Python + FastAPI + SQLAlchemy + Alembic。
- 数据库：PostgreSQL。
- 异步任务：Redis + RQ。
- 部署：Docker Compose。
- 架构：模块化单体，保持简单高效可用。

## 阶段 0：客户数据盘点与试点范围确认

**对应路线图阶段：** `docs/ai-native-enterprise-data-platform-roadmap.md` 的“阶段 0：客户数据盘点与试点范围确认”。

**对应产品建设方案：** `docs/ai-native-enterprise-data-platform-solution.md` 的产品定位、目标客户、典型问题和中小制造企业复杂度边界；`docs/ai-native-enterprise-data-platform-foundation.md` 的基础建设定位和准入标准。

**对应 PRD/规格：** `docs/ai-native-enterprise-data-platform-foundation-prd.md` 的产品目标、产品边界、用户角色、验收标准和研发输入要求。阶段 0 的输出不是新增系统功能，而是形成阶段 1 研发输入。

**阶段目标：** 在开发前确认先接哪些数据、服务哪些业务对象、解决哪些高价值问题，避免一开始范围过大。

### T00：客户数据盘点与试点范围确认

**建议角色：** 产品 / 数据顾问 / 实施

**依赖：** 无

**范围：**

- 盘点首批数据源：ERP、MES、WMS、Excel、共享盘或其他系统。
- 确认首批业务对象：客户、订单、产品、物料、库存、设备、工序等。
- 确认首批高价值问题：订单进度、交付风险、库存异常、质量异常、设备异常等。
- 确认数据负责人、业务负责人和系统负责人。
- 确认字段敏感等级、权限边界和 Agent 可用边界。
- 输出阶段 1 的接入范围，不追求全量系统和全量历史数据。

**交付物：**

- 数据源盘点表。
- 业务对象清单。
- 试点问题清单。
- 字段敏感等级和权限初稿。
- 阶段 1 建设范围确认记录。

**验收标准：**

- 首批数据源、业务对象和试点问题已确认。
- 每个数据源都有业务负责人和技术负责人。
- 阶段 1 不做什么已经写清楚。
- 团队可基于盘点结果开始阶段 1 研发。

### 阶段 3 验收边界

- Agent 可以查询数据目录、字段、质量状态、指标口径和业务关系路径。
- Agent 回答必须展示数据来源、工具调用、口径和不确定性。
- Agent 不允许绕过权限读取数据。
- Agent 不允许直接写回 ERP、MES、采购、财务等核心系统。
- 指标和规则只做当前试点够用的轻量管理，不建设完整 BI 指标平台。

## 阶段 4：Agent 建议与协同闭环

**对应路线图阶段：** `docs/ai-native-enterprise-data-platform-roadmap.md` 的“阶段 4：Agent 建议与协同闭环”。

**对应产品建设方案：** `docs/ai-native-enterprise-data-platform-solution.md` 的 Agent 行动分级和受控协同理念；`docs/ai-native-enterprise-data-platform-technical-design.md` 的建议与协同技术边界。

**对应 PRD/规格：** `docs/ai-native-enterprise-data-platform-foundation-prd.md` 的 Agent 回答要求、禁止行为、通知日志、权限审计和总体验收要求。阶段 4 的产品边界是建议、预警、待办、人工确认和审计闭环，不包含自动写回核心业务系统。

**阶段目标：** 让 Agent 从“回答问题”进入“提出建议并推动协同”，但所有动作保持可解释、可确认、可追踪。

### 阶段 4 验收边界

- Agent 可以基于规则、质量问题、关系路径和查询结果生成建议或预警草稿。
- 建议必须包含依据、来源、影响对象、置信度或不确定性。
- 建议只能进入人工确认流程，不能自动修改核心系统数据。
- 人工确认、驳回、备注和审计记录完整可查。
- 当前阶段不建设复杂审批流、工作流引擎或跨系统自动执行。
