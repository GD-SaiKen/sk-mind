## 接入信息

| 项目 | 内容 |
|---|---|
| **接入方式** | 数据库只读连接 → 定时同步 |
| **同步策略** | 全量同步（初始）+ 增量同步（后续，依赖时间戳字段） |
| **接入频率建议** | 日间低峰期（T+1 模式），核心表可支持小时级 |
| **连接要求** | 只读账号，最小权限，屏蔽 DROP/ALTER/DELETE |
| **Schema 变化风险** | 中 — E10 版本升级可能新增/变更表和字段 |

## 风险与限制

| 风险项 | 说明 | 影响 |
|---|---|---|
| **表量巨大** | 1,989 张表，无法逐一梳理 | 需分批推进，优先核心域核心表 |
| **字段注释缺失** | 大部分字段无数据库注释 | 字段业务含义需 AI 推断 + 人工确认 |
| **日志/临时表混杂** | 存在大量 _LOG、_TEMPLATE、_BAK 后缀表 | 需筛选过滤，Agent 不可用 |
| **跨域外键复杂** | 表间关系分散在 SQL 中，无集中外键约束 | 关系图谱需从代码/文档推断 |
| **版本差异** | E10 版本升级可能变更 Schema | 梳理成果需备注版本号，版本升级后需增量更新 |
| **数据量** | 部分主表（如 MO、SALE）行数可能百万级 | 接入时需考虑分批和增量策略 |
| **权限风险** | ERP 数据库含财务敏感数据 | 只读接入，字段级脱敏，Agent 权限继承 |
