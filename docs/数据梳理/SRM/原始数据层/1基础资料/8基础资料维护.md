此API接口是查询平台基础资料维护数据，包括1个API接口：

###   [检验不良原因列表查询](https://openapi.xiekeyun.com/doc/docpull/archives/basicData.html#1-检验不良原因列表查询)

​     **调用场景：**

​       此API接口是查询检验不良原因数据；

## 1 检验不良原因列表查询

此API接口是查询检验不良原因数据。

## 1.1 请求

#### 1.1.1 请求地址

https://openapi.xiekeyun.com/basicData/getBadReasonList.json

#### 1.1.2 请求公传参数（commonParam）

**请求时对应的公传参数定义如【[公传参数](https://openapi.xiekeyun.com/doc/doccommon/description.html#公传参数)】所述**

#### 1.1.3 业务请求参数（body）

| 参数名称 | 参数描述          | 参数类型   | 说明                                                         | 要求     |
| -------- | ----------------- | ---------- | ------------------------------------------------------------ | -------- |
| erpCode  | 请求者用户ERP帐号 | **String** | 对应数据操作公司中，员工档案里“ERP帐户”的值，主要做数据鉴权使用，创建此员工写入的数据 | **必填** |

#### 1.1.4 请求参数示例

Demo

```
{
    "commonParam": {
        "operateCompanyCode": "27967281",
        "timestamps": 1669365639,
        "sign": "042a385538ccb77aa50c44f15c759f49",
        "appKey": "70d4e8e9f384b9e266807e5d79a32d60",
        "ownerCompanyCode": "27967281",
        "version": "1.0"
    },
    "body": {
        "erpCode": "ds"
    }
}
    
```

#### 1.1.5 说明

1. 本接口访问频率不能低于 **5** 分钟；

### 1.2 结果返回

#### 1.2.1 描述

结果数据以Json格式返回

#### 1.2.2 返回结果

#### 1.2.2.1 返回结果返回结果字段定义

| 字段名称  | 描述                                                         | 类型         |
| --------- | ------------------------------------------------------------ | ------------ |
| result    | 返回结果状态 1:成功；9:失败                                  | **Number**   |
| errorCode | 接口错误编码（失败时有值），详细参考“[**错误编码规范定义**](https://openapi.xiekeyun.com/doc/doccommon/description.html#错误编码规范定义)” | **String**   |
| errorMsg  | 接口错误信息（失败时有值）                                   | **String**   |
| dataList  | 已集成的采购组织+工厂列表数据                                | **JSON数组** |

#### 1.2.2.2 dataList中一个Json数据明细字段定义

| 字段名称         | 描述                            | 类型       |
| ---------------- | ------------------------------- | ---------- |
| verifyReasonCode | 原因编码（系统生成）            | **String** |
| verifyReasonName | 原因名称                        | **String** |
| deductRate       | 扣款率                          | **Number** |
| badDegree        | 不良程度 1:主要不良，2:次要不良 | **Number** |

此API接口是查询平台基础分类数据

###   [分类管理设定查询](https://openapi.xiekeyun.com/doc/docpull/archives/customCategory.html#1-分类管理设定查询)

​     **调用场景：**

​       此API接口是查询平台分类管理设定数据；

## 1 分类管理设定查询

此API接口是查询分类维护数据。

## 1.1 请求

#### 1.1.1 请求地址

https://openapi.xiekeyun.com/customCategory/queryByType.json

#### 1.1.2 请求公传参数（commonParam）

**请求时对应的公传参数定义如【[公传参数](https://openapi.xiekeyun.com/doc/doccommon/description.html#公传参数)】所述**

#### 1.1.3 业务请求参数（body）

| 参数名称         | 参数描述                                 | 参数类型       | 说明                                                         | 要求     |
| ---------------- | ---------------------------------------- | -------------- | ------------------------------------------------------------ | -------- |
| erpCode          | 请求者用户ERP帐号                        | **String**     | 对应数据操作公司中，员工档案里“ERP帐户”的值，主要做数据鉴权使用，创建此员工写入的数据 | **必填** |
| settingFlag      | 是否按管理分类设定返回 0-否 1-是 默认:是 | **Number**     |                                                              | **选填** |
| categoryTypeList | 分类所属类型                             | **String数组** | 类型值定义:2000（ERP）; 2001（物料分类1）; 2002（物料分类2）; 2003（物料分类3） | **选填** |

#### 1.1.4 请求参数示例

Demo

```
            {
                "commonParam": {
                    "appKey": "70d4e8e9f384b9e266807e5d79a32d60",
                    "version": "1.0",
                    "ownerCompanyCode": "27967281",
                    "operateCompanyCode": "27967281",
                    "sign": "5fe9bc9f6c200b1b65c62e738f47ccf9",
                    "timestamps": "1775125248"
                },
                "body": {
                    "erpCode": "69999"
                }
            }
    
```

#### 1.1.5 说明

1. 本接口访问频率不能低于 **5** 分钟；

### 1.2 结果返回

#### 1.2.1 描述

结果数据以Json格式返回

#### 1.2.2 返回结果

#### 1.2.2.1 返回结果返回结果字段定义

| 字段名称  | 描述                                                         | 类型         |
| --------- | ------------------------------------------------------------ | ------------ |
| result    | 返回结果状态 1:成功；9:失败                                  | **Number**   |
| errorCode | 接口错误编码（失败时有值），详细参考“[**错误编码规范定义**](https://openapi.xiekeyun.com/doc/doccommon/description.html#错误编码规范定义)” | **String**   |
| errorMsg  | 接口错误信息（失败时有值）                                   | **String**   |
| dataList  | 分类数据                                                     | **JSON数组** |

#### 1.2.2.2 dataList 中一个 Json 数据明细字段定义

| 字段名称                | 描述                                                         | 类型          |
| ----------------------- | ------------------------------------------------------------ | ------------- |
| categoryType            | 分类所属类型（2000:物料 erp 分类; 2001:物料分类1; 2002:物料分类2; 2003:物料分类 3） | **String**    |
| categoryCode            | 分类编码                                                     | **String**    |
| categoryName            | 分类名称                                                     | **String**    |
| parentCode              | 类别编码                                                     | **String**    |
| parentName              | 类别名称（定义说明：01:生产物料;02:工程服务;03:物流服务;04:委外生产;05:设备仪器;06:IT服务;07:综合服务;08:非生产类物质） | **String**    |
| purchaseGroupCode       | 采购组编码                                                   | **String**    |
| purchaseGroupName       | 采购组名称                                                   | **String**    |
| valuationQtyFlag        | 计价数量设置：0-Off，1-On                                    | **Number**    |
| minPackingFlag          | 是否按最小包装量出货，0:否，1-是                             | **Number**    |
| purchaseStrategy        | 采购策略 (1:核定供应商;2:询价招标;3:内部商城;4:商城选购;5:自选供应商;6:自由采购;7:项目招标) | **Number**    |
| inqSourceBuyerGroupCode | 寻源采购组编码                                               | **String**    |
| inqSourceBuyerGroupName | 寻源采购组名称                                               | **String**    |
| inqSourceBuyerList      | 寻源采购员列表（Json 数组）                                  | **Json 数组** |
| buyerList               | 采购员列表（Json 数组）                                      | **Json 数组** |
| inspectorList           | 检验员列表（Json 数组）                                      | **Json 数组** |
| storekeeperList         | 仓管员列表（Json 数组）                                      | **Json 数组** |

#### 1.2.2.3 inqSourceBuyerList、buyerList、inspectorList、storekeeperList 字段结构明细字段定义

| 字段名称        | 描述         | 类型       |
| --------------- | ------------ | ---------- |
| employeeNumber  | 工号         | **String** |
| employeeName    | 员工姓名     | **String** |
| erpAccount      | ERP 账号     | **String** |
| erpEmployeeCode | ERP 员工编码 | **String** |
