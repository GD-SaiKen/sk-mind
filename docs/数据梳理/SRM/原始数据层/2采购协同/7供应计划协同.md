- [直运看板](https://openapi.xiekeyun.com/doc/docpull/order/directDeliveryBoard-desc.html)
- [供应商库存看板](https://openapi.xiekeyun.com/doc/docpull/order/prodStockBoard-desc.html)

## 1 排程列表查询

  此API接口查询排程单号列表

### 1.1 请求

#### 1.1.1 请求地址

  https://openapi.xiekeyun.com/schedule/getList.json

#### 1.1.2 请求公传参数（commonParam）

  **请求时对应的公传参数定义如【[公传参数](https://openapi.xiekeyun.com/doc/doccommon/description.html#公传参数)】所述**

#### 1.1.3 业务请求参数（body）

| 参数名称         | 参数描述                                                     | 参数类型       | 说明                                                         | 要求     |
| ---------------- | ------------------------------------------------------------ | -------------- | ------------------------------------------------------------ | -------- |
| refBatchNo       | 来源排程单号                                                 | **String**     | 来源排程单号                                                 | **选填** |
| startDate        | 发布时间                                                     | **Number**     | 排程发布、答交、差异确认的开始时间对应的时间戳数值; 查询时为开区间（即 >startDate） | **必填** |
| endDate          | 发布时间                                                     | **Number**     | 排程发布或者答交的结束时间对应的时间戳数值; 查询时为闭区间（即 <=endDate）(startDate,endDate]最大时间区间为120分钟 | **必填** |
| erpCode          | 请求者用户ERP帐号                                            | **String**     | 对应数据操作公司中，员工档案里“ERP帐户”的值，主要做数据鉴权使用，返回此员工有权限的数据 | **必填** |
| selectType       | 列表查询类型                                                 | **Number**     | 0:只返回采购方排程差异确认自动产生的排程单列表; 1:返回发布时间范围内所有排程单号列表默认值：0 | **选填** |
| statusList       | 状态 已发布：1,已发布；2,部分答交*无差异；3,有差异*未确认；4,全部答交；5 已失效；6 退回；7 部分退回；8作废 | **Number数组** | 查询时，如果传值，返回对应状态的单据；不传值，默认返回[1,2,3,4,5]状态单据 | **选填** |
| schedulePlanType | 计划类型                                                     | **Number**     | 1:交货计划; 2:预测计划：                                     | **选填** |
| oldBatchTypes    | 子排程计划类型                                               | **Number数组** | 0:交货计划; 1:更新计划：2：调整计划；3：增量计划；4：排程变更 | **选填** |

#### 1.1.4 请求参数示例

Demo

```
{
  "commonParam":{
    "appKey":"3da512e5eb0aa2fa27b6c3ffc638da22",
    "version":"1.0",
    "ownerCompanyCode":"91904378",
    "operateCompanyCode":"91904378",
    "sign":"dd2e0c02d1e0e71026cf28c56cc0e531",
    "timestamps":"1572945473"
  },
  "body":{
    "erpCode":"liuzy",
    "startDate":1572686269000,
    "endDate":1572693469000,
    "refBatchNo":"3020191102615540"
  }
}
    
```

#### 1.1.5 说明

  \1. 本接口访问频率不能低于 **1** 分钟；

  \2. 一次请求中开始结束时间范围不能大于 **120** 分钟；

  \3. 查询数据时，不能查询 **30** 天之前的数据。

### 1.2 结果返回

#### 1.2.1 描述

  结果数据以Json格式返回

#### 1.2.2 返回结果

#### 1.2.2.1 返回结果公共字段定义

| 字段名称  | 描述                                                         | 类型         |
| --------- | ------------------------------------------------------------ | ------------ |
| result    | 返回结果状态 1:成功；9:失败                                  | **Number**   |
| errorCode | 接口错误编码（失败时有值），详细参考“[**错误编码规范定义**](https://openapi.xiekeyun.com/doc/doccommon/description.html#错误编码规范定义)” | **String**   |
| errorMsg  | 接口错误信息（失败时有值）                                   | **String**   |
| dataList  | 排程单号数据列表                                             | **JSON数组** |

#### 1.2.2.2 dataList中一个Json数据明细字段定义

| 字段名称         | 描述                                                         | 类型       |
| ---------------- | ------------------------------------------------------------ | ---------- |
| batchNo          | 排程批次号                                                   | **String** |
| oldBatchNo       | 父排程单号                                                   | **String** |
| refBatchNo       | 来源排程单号                                                 | **String** |
| status           | 状态：1-已发布;2-部分答交*无差异;3-有差异*未确认;4-全部答交 ;5-已失效 | **Number** |
| publishName      | 发布人员工姓名                                               | **String** |
| publishTime      | 发布时间对应的时间戳(毫秒)                                   | **Number** |
| schedulePlanType | 计划类型 1:交货计划; 2:预测计划：                            | **Number** |

## 3 排程发布结果查询

  此API接口查询排程发布处理结果

### 3.1 请求

#### 3.1.1 请求地址
