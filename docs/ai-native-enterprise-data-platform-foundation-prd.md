## 文档说明

本文档是《AI 原生企业数据平台基础建设方案》和《AI 原生企业数据平台基础功能清单与验收检查表》的产品需求说明，用于描述基础平台阶段的用户角色、信息架构、核心流程、模块需求、状态处理和验收标准。

## 1. 产品目标

基础平台的产品目标是先完成 AI 原生企业数据平台的底座能力，让企业数据能进入平台、被管理、被治理、被语义描述，并能被 Agent 在受控边界内查询和解释。

基础平台第一阶段不追求完成某个具体业务场景闭环，也不追求让 Agent 自动做复杂业务决策。它重点解决以下问题：

- 企业有哪些数据源。
- 哪些数据已经接入平台。
- 数据从哪里来，什么时候进入，质量如何。
- 用户和 Agent 能否在权限范围内查询数据。
- 数据是否有目录、字段说明、负责人和质量状态。
- 核心数据是否能够映射到轻量语义模型。
- 业务对象之间的关系是否能记录为 BusinessGraphEdge。
- Agent 是否能说明数据来源、质量状态和不确定性。

基础平台的成功标志不是功能数量多，而是形成一条可信链路：

```text
数据源登记
→ 数据接入
→ 平台数据库入库
→ Raw / Clean / Serving 分层
→ 数据目录和质量标记
→ 权限与审计
→ 轻量语义映射
→ 业务关系图谱
→ Agent 受控查询和解释
```

## 2. 产品边界

### 2.1 本阶段包含

基础平台阶段包含以下产品能力：

| 能力域 | 本阶段范围 |
| --- | --- |
| 数据源管理 | 登记、分类、负责人、接入方式、状态管理 |
| 数据接入 | 接入任务、批次、日志、错误、文件导入、Excel 批量导入 |
| 数据分层 | Raw、Clean、Serving 的基础管理和展示 |
| 通用数据表 | 表列表、表结构、样例、字段说明、负责人、Agent 可用标识 |
| 数据目录 | 数据集、字段、来源、更新时间、质量状态、权限等级 |
| 数据质量 | 基础质量规则、执行结果、质量状态、问题清单 |
| 权限审计 | 用户、角色、部门、数据集权限、敏感字段、访问审计 |
| 语义框架 | BusinessObject、ObjectProperty、SemanticRelation、DataMapping、ObjectActionPolicy |
| 业务关系图谱 | BusinessGraphEdge 存储、查询、来源、可信度、确认状态 |
| Agent 数据服务 | 数据目录检索、权限检查、受控查询、来源说明、质量提示、调用审计 |
| 管理控制台 | 基础运营入口和状态总览 |

### 2.2 本阶段不包含

基础平台阶段不包含：

- 全企业所有系统一次性接入。
- 全量历史 Excel 自动理解。
- 重型知识图谱平台。
- RDF、OWL、三元组存储和复杂推理机。
- Agent 自动写回 ERP、MES、财务等核心系统。
- 完整业务场景闭环。
- 完整指标体系和管理驾驶舱。
- 复杂工作流引擎。

这些能力可以在业务场景验证或后续阶段逐步增强。

## 3. 用户角色

### 3.1 角色总览

| 角色 | 主要职责 | 典型操作 |
| --- | --- | --- |
| 平台管理员 | 管理平台配置、用户、权限、数据源和基础运营 | 配置用户角色、查看任务状态、处理异常 |
| 数据负责人 | 负责某类数据源、数据表、字段说明和质量问题 | 登记数据源、确认字段含义、处理质量问题 |
| 业务用户 | 查询数据、查看目录、使用 Agent 获取基础解释 | 检索数据、查看表结构、向 Agent 提问 |
| 语义建模人员 | 维护业务对象、属性、关系、映射和对象行动策略 | 创建对象、配置映射、确认关系 |
| 审计人员 | 查看数据访问、权限变更和 Agent 调用记录 | 检索审计日志、追踪访问行为 |
| Agent 使用者 | 通过自然语言查询数据、理解来源和质量限制 | 询问有哪些数据、查询某张表、查看来源 |

### 3.2 角色权限原则

基础平台遵循以下权限原则：

- 人和 Agent 使用同一套权限边界。
- Agent 不能绕过当前用户权限访问数据。
- 数据负责人可以维护自己负责的数据说明和质量处理状态。
- 语义建模人员可以维护语义对象和映射，但不能默认访问所有业务数据。
- 审计人员可以查看日志，但不默认拥有业务数据明细访问权限。
- 平台管理员拥有配置权限，但高敏数据仍应支持脱敏和审计。

| 数据质量 | 质量规则、执行结果、问题清单、质量状态 |
| 权限审计 | 用户、角色、部门、数据权限、敏感字段、访问审计 |

### 4.2 首页概览

首页用于让平台管理员和数据负责人快速了解基础平台状态。

首页建议展示：

- 数据源总数、正常数、异常数。
- 接入任务最近执行状态。
- 数据表数量和最近更新时间。
- 质量检查通过、警告、异常数量。
- 敏感字段数量。
- 已开放给 Agent 的数据集数量。
- 语义对象数量。
- BusinessGraphEdge 数量。
- Agent 查询次数和失败次数。

首页不承担复杂分析职责，只提供基础运营概览和异常入口。

## 5. 核心用户流程

### 5.1 数据接入到 Agent 可用流程

这是基础平台最核心的流程。

```text
登记数据源
→ 创建接入任务
→ 执行接入
→ 数据进入 Raw 层
→ 生成 Clean 层数据和质量标记
→ 形成通用数据表
→ 补充字段说明和负责人
→ 配置数据目录和权限
→ 标记是否允许 Agent 查询
→ 配置语义对象和字段映射
→ Agent 在权限范围内查询和说明来源
```

该流程的验收标准：

- 每一步都有可查看状态。
- 失败时能定位原因。
- 数据来源和批次可追踪。
- Agent 只能访问允许且有权限的数据。

### 5.2 Excel 批量导入流程

```text
登记 Excel 数据源
→ 上传少量模板样本
→ 配置或确认解析规则
→ 执行样本导入
→ 查看字段和样例数据
→ 确认无误
→ 配置本地目录或共享盘目录
→ 批量导入历史文件
→ 查看成功和失败文件清单
→ 失败文件修正后重新导入
```

该流程的原则：

- 不要求用户逐个上传大量历史文件。
- 每类模板先确认样本，再批量处理。
- 导入失败不阻塞已成功文件入库。
- 所有文件保留路径、文件名、模板类型和导入批次。

### 5.3 数据质量处理流程

```text
选择数据集
→ 配置或选择质量规则
→ 执行质量检查
→ 查看问题清单
→ 标记问题状态
→ 分派给数据负责人
→ 修正数据或确认可接受
→ 更新质量状态
→ Agent 查询时读取质量状态
```

基础阶段不要求一次性修复所有质量问题，但必须让问题可见、可定位、可说明。

### 5.4 语义映射流程

```text
创建业务对象
→ 创建对象属性
→ 定义对象关系
→ 选择数据表和字段
→ 建立 DataMapping
→ 标记映射可信度
→ 配置对象行动策略
→ 开放给 Agent 语义检索
```

基础阶段可以先定义少量通用对象，不要求完整业务域建模。

### 5.5 Agent 查询流程

```text
用户提出问题
→ Agent 识别查询意图
→ 检查用户权限
→ 检索数据目录或语义模型
→ 定位允许访问的数据集
→ 查询数据或关系路径
→ 读取质量状态和来源信息
→ 生成回答
→ 记录工具调用和审计日志
```

Agent 回答必须尽量包含：

- 直接回答。
- 使用的数据源或数据集。
- 更新时间。
- 质量状态。
- 权限限制。
- 不确定性说明。

## 6. 通用页面与交互原则

### 6.1 列表页

所有列表页应支持：

- 关键词搜索。
- 状态筛选。
- 类型筛选。
- 负责人筛选。
- 更新时间排序。
- 批量选择。
- 查看详情。

列表页需要清晰展示状态，不让用户进入详情页才知道是否异常。

### 6.2 详情页

详情页应包含：

- 基本信息。
- 状态信息。
- 来源信息。
- 负责人。
- 最近更新时间。
- 关联对象或任务。
- 操作记录。
- 错误或警告信息。

### 6.3 状态展示

状态需要统一表达，建议包括：

| 状态 | 使用场景 |
| --- | --- |
| 未配置 | 数据源、任务、规则或映射尚未配置完成 |
| 待确认 | 需要用户确认样本、字段、关系或质量问题 |
| 正常 | 当前对象可用 |
| 警告 | 存在非阻塞问题 |
| 异常 | 存在阻塞问题 |
| 停用 | 当前对象不再使用 |

### 6.4 操作反馈

所有关键操作都需要明确反馈：

- 操作成功。
- 操作失败。
- 部分成功。
- 需要确认。
- 无权限。
- 数据不存在。
- 数据质量不足。

## 7. 数据源管理需求

### 7.1 目标

数据源管理用于登记企业内部所有可接入或待接入的数据来源，建立数据接入的基础台账。

### 7.2 页面

| 页面 | 说明 |
| --- | --- |
| 数据源列表 | 展示所有数据源、类型、负责人、状态和接入方式 |
| 数据源详情 | 展示数据源基础信息、接入配置摘要、任务、日志和风险 |
| 新增数据源 | 引导用户登记数据源 |
| 编辑数据源 | 修改负责人、状态、说明、标签和接入方式 |

### 7.3 功能需求

| 优先级 | 功能 | 说明 |
| --- | --- | --- |
| P0 | 新增数据源 | 支持登记 ERP、MES、采购、考勤、数据库、API、Excel、共享盘等来源 |
| P0 | 编辑数据源 | 支持修改名称、类型、负责人、说明和状态 |
| P0 | 数据源状态 | 支持未接入、接入中、正常、异常、停用 |
| P0 | 负责人管理 | 支持配置业务负责人和技术负责人 |
| P0 | 接入方式记录 | 记录数据库、API、文件、Excel、共享盘等接入方式 |
| P1 | 标签管理 | 支持部门、业务域、敏感等级等标签 |
| P1 | 风险说明 | 记录接口限制、权限限制、文件质量等风险 |

### 7.4 验收标准

- 可以创建和编辑第一批数据源。
- 数据源有类型、负责人、接入方式和状态。
- 异常数据源可以记录原因。
- 数据源可以关联接入任务。

## 8. 数据接入任务需求

### 8.1 目标

数据接入任务用于将数据源中的数据导入平台数据库，并记录任务状态、执行批次、日志、错误和数据量。

### 8.2 页面

| 页面 | 说明 |
| --- | --- |
| 接入任务列表 | 展示任务名称、数据源、状态、最近执行时间和结果 |
| 任务详情 | 展示任务配置、批次、日志、错误和产出数据表 |
| 批次详情 | 展示某次执行的时间、数据量、文件数、成功和失败结果 |
| 错误清单 | 展示失败文件、失败记录和错误原因 |

### 8.3 功能需求

| 优先级 | 功能 | 说明 |
| --- | --- | --- |
| P0 | 创建任务 | 为数据源创建接入任务 |
| P0 | 执行任务 | 支持手动执行任务 |
| P0 | 批次记录 | 每次执行生成批次号和执行结果 |
| P0 | 日志查看 | 查看任务执行日志和错误原因 |
| P0 | 失败重试 | 失败后可以重新执行 |
| P0 | 文件导入 | 支持 CSV 和 Excel 文件导入 |
| P0 | Excel 样本确认 | 每类 Excel 模板先确认样本 |
| P0 | Excel 批量导入 | 支持按目录或共享盘路径批量导入 |
| P1 | 数据库同步 | 支持数据库表同步配置 |
| P1 | API 拉取 | 支持 API 数据拉取配置 |
| P1 | 共享盘扫描 | 支持共享盘目录扫描和增量识别 |

### 8.4 状态

| 状态 | 说明 |
| --- | --- |
| 未运行 | 任务已创建但尚未执行 |
| 运行中 | 任务正在执行 |
| 成功 | 本次任务全部成功 |
| 部分成功 | 部分数据成功，部分失败 |
| 失败 | 本次任务未成功产出数据 |
| 已停用 | 任务暂停使用 |

### 8.5 验收标准

- 接入任务可以生成批次和日志。
- 文件和 Excel 可以导入平台数据库。
- Excel 支持样本确认后批量导入。
- 失败文件和失败记录可以定位。
- 接入产出能进入数据表管理和数据目录。

## 9. 平台数据库与数据分层需求

### 9.1 目标

平台数据库与数据分层用于承载接入数据，并让数据从原始状态逐步进入可查询、可治理、可语义映射的状态。

### 9.2 分层要求

| 层级 | 产品要求 |
| --- | --- |
| Raw 层 | 保留原始字段、原始值、来源、文件路径、批次和导入时间 |
| Clean 层 | 保存基础清洗、类型转换、质量标记和去重结果 |
| Serving 层 | 面向查询、语义映射和 Agent 使用提供稳定访问入口 |

### 9.3 功能需求

| 优先级 | 功能 | 说明 |
| --- | --- | --- |
| P0 | Raw 数据查看 | 有权限用户可以查看原始数据样例和来源信息 |
| P0 | Clean 数据查看 | 有权限用户可以查看清洗后数据和质量状态 |
| P0 | Serving 视图查看 | 展示面向查询和 Agent 使用的数据入口 |
| P0 | 来源追踪 | 可以从数据追踪到来源系统、表、文件、任务和批次 |
| P0 | 批次筛选 | 支持按批次查看数据 |
| P1 | 版本记录 | 记录数据结构和处理规则变化 |

### 9.4 验收标准

- 接入数据可以进入 Raw 层。
- Clean 层可以展示质量标记后的数据。
- Serving 层可以作为查询和 Agent 使用入口。
- 任意数据可以追踪来源和批次。

## 10. 通用数据表管理需求

### 10.1 目标

通用数据表管理用于承接基础阶段接入的数据，让数据表可查看、可描述、可授权、可映射和可开放给 Agent。

### 10.2 页面

| 页面 | 说明 |
| --- | --- |
| 数据表列表 | 展示表名、来源、数据量、负责人、更新时间、质量状态 |
| 数据表详情 | 展示字段、样例数据、来源、批次、权限和 Agent 可用状态 |
| 字段详情 | 展示字段类型、说明、来源、敏感标记和映射状态 |

### 10.3 功能需求

| 优先级 | 功能 | 说明 |
| --- | --- | --- |
| P0 | 表列表 | 展示平台已有通用数据表 |
| P0 | 表结构查看 | 展示字段名、类型、样例值、空值情况 |
| P0 | 样例数据查看 | 支持查看有限样例数据 |
| P0 | 字段说明维护 | 支持补充字段业务说明 |
| P0 | 表负责人 | 支持配置数据表负责人 |
| P0 | Agent 可用标识 | 控制数据表是否允许 Agent 查询 |
| P1 | 字段显示名 | 支持设置业务可读显示名 |
| P1 | 表标签 | 支持按业务域、部门、系统打标签 |

### 10.4 验收标准

- 接入数据可以形成数据表。
- 数据表可以查看结构和样例。
- 字段可以维护业务说明。
- 数据表可以控制是否开放给 Agent。

## 11. 数据目录与元数据需求

### 11.1 目标

数据目录用于让人和 Agent 快速理解平台中有哪些数据、数据从哪里来、谁负责、质量如何、是否可用。

### 11.2 页面

| 页面 | 说明 |
| --- | --- |
| 数据集目录 | 展示数据集、来源、负责人、质量、权限和 Agent 可用状态 |
| 字段目录 | 展示字段、含义、类型、敏感标记、映射状态 |
| 数据集详情 | 展示数据集说明、字段、来源、血缘、质量和使用记录 |

### 11.3 功能需求

| 优先级 | 功能 | 说明 |

### 12.2 页面

| 数据集质量详情 | 展示某数据集的质量状态和问题分布 |

### 12.3 功能需求

| 优先级 | 功能 | 说明 |
| --- | --- | --- |
| P0 | 完整性检查 | 检查关键字段为空 |
| P0 | 唯一性检查 | 检查主键或候选键重复 |
| P0 | 格式检查 | 检查日期、金额、编码格式 |
| P0 | 类型检查 | 检查字段值与字段类型是否匹配 |
| P0 | 枚举检查 | 检查状态值是否在允许范围内 |
| P0 | 质量状态 | 形成未检查、通过、警告、异常、待确认状态 |
| P0 | 问题清单 | 展示问题数量、问题字段、样例和影响范围 |

### 12.4 验收标准

- 检查结果可以查看。

## 16. Agent 数据服务需求

### 16.1 目标

Agent 数据服务用于让 Agent 在受控范围内使用平台数据。基础阶段 Agent 主要承担查询、发现、解释来源和说明限制，不承担自动业务写入。

### 16.2 Agent 可用工具

| 工具 | 作用 |
| --- | --- |
| 数据目录检索工具 | 查询数据集、字段、来源和负责人 |
| 权限检查工具 | 检查当前用户是否可以访问目标数据 |
| 受控数据查询工具 | 查询允许访问的数据表或视图 |
| 质量状态查询工具 | 查询数据质量状态和问题摘要 |
| 语义检索工具 | 查询对象、属性、关系和映射 |
| 图谱查询工具 | 查询对象实例关系路径 |
| 审计记录工具 | 记录问题、工具调用、数据和回答 |

### 16.3 回答要求

Agent 基础回答应包含：

- 直接回答。
- 使用的数据集或字段。
- 数据来源。
- 更新时间。
- 质量状态。
- 权限限制。
- 不确定性说明。

### 16.4 禁止行为

基础阶段 Agent 不允许：

- 绕过用户权限查询数据。
- 直接访问未知数据库或文件。
- 自动写回 ERP、MES、采购、考勤、财务等核心系统。
- 在数据质量异常时给出确定性结论。
- 隐藏数据来源和口径限制。

### 16.5 验收标准

- Agent 可以查询数据目录和字段说明。
- Agent 可以在权限范围内查询数据。
- Agent 回答包含来源、更新时间和质量状态。
- Agent 调用有审计记录。
- Agent 不自动执行高风险写入动作。

## 18. 状态、异常与空数据处理

### 18.1 空数据状态

空数据状态需要说明下一步操作：

| 页面 | 空状态提示 |
| --- | --- |
| 数据源列表 | 当前还没有数据源，请先登记一个数据源 |
| 接入任务列表 | 当前还没有接入任务，请先为数据源创建任务 |
| 数据表列表 | 当前还没有接入数据，请先执行接入任务 |
| 数据目录 | 当前还没有可用数据集，请先完成数据接入 |
| 语义对象 | 当前还没有业务对象，请先创建通用对象 |
| 关系图谱 | 当前还没有关系边，请先配置关系映射或导入关系 |
| Agent 审计 | 当前还没有 Agent 调用记录 |

### 18.2 异常状态

常见异常包括：

| 异常 | 处理要求 |
| --- | --- |
| 数据源不可连接 | 展示失败原因、最近检测时间和负责人 |
| 接入任务失败 | 展示失败批次、错误日志和重试入口 |
| Excel 解析失败 | 展示文件名、行列位置、错误原因和修正建议 |
| 数据质量异常 | 展示问题字段、样例数据和影响范围 |
| 权限不足 | 告知用户无权限，并提供申请或联系负责人入口 |
| Agent 查询失败 | 展示失败原因，例如无权限、无数据、质量异常或工具失败 |

### 18.3 部分成功状态

接入、质量检查和批量导入可能出现部分成功。产品上需要明确展示：

- 成功数量。
- 失败数量。
- 跳过数量。
- 失败原因。
- 是否可重试。
- 哪些数据已经入库。
- 哪些数据不能被 Agent 使用。

## 19. 通知与日志需求

### 19.1 通知

基础阶段建议支持以下通知：

- 接入任务失败。
- Excel 批量导入完成。
- 数据质量异常。
- 权限变更。
- 语义映射待确认。
- 关系边待确认。
- Agent 查询失败率异常。

通知可以先在平台内展示，后续再扩展到企业微信、钉钉、邮件或 OA。

### 19.2 日志

基础阶段需要记录：

- 数据源变更日志。
- 接入任务执行日志。
- 数据导入批次日志。
- 数据质量检查日志。
- 权限变更日志。
- 语义模型变更日志。
- BusinessGraphEdge 生成和确认日志。
- Agent 工具调用日志。

日志需要支持按时间、用户、对象类型、状态和关键词查询。

## 20. 基础阶段不做什么

基础阶段不做以下事项：

| 不做事项 | 说明 |
| --- | --- |
| 全系统一次性接入 | 先支持多种接入能力，再分批接入实际系统 |
| 自动理解所有 Excel | 先按模板样本确认后批量导入 |
| 重型知识图谱 | 先做轻量语义和 BusinessGraphEdge |
| 自动写回核心系统 | Agent 不自动修改 ERP、MES、财务等数据 |
| 完整业务闭环 | 业务验证阶段再选择主线推进 |
| 完整指标驾驶舱 | 指标在业务验证中逐步沉淀 |
| 复杂工作流 | 基础阶段只做基础确认、日志和审计 |

## 21. 验收标准

### 21.1 总体验收

基础平台进入业务场景验证前，需要满足：

| 验收域 | 标准 |
| --- | --- |
| 数据接入 | 第一批数据源已接入，任务有批次、日志和错误记录 |
| 数据分层 | Raw、Clean、Serving 基础链路可用 |
| 数据目录 | 数据集、字段、来源、负责人、更新时间可查看 |
| 数据质量 | 基础质量规则可执行，质量状态可查看 |
| 权限审计 | 用户、角色、数据集权限、敏感字段和访问审计可用 |
| 通用数据表 | 接入数据可查看、描述、授权和开放给 Agent |
| 语义框架 | 可定义对象、属性、关系、映射和对象行动策略 |
| 关系图谱 | 可存储和查询基础 BusinessGraphEdge |
| Agent 数据服务 | Agent 可受控查询数据并说明来源、质量和限制 |
| 管理控制台 | 基础配置和运营入口可用 |

### 21.2 准入判断

如果以下能力没有完成，不建议进入业务场景验证：

- 数据接入。
- 数据分层。
- 数据目录。
- 数据质量状态。
- 权限和审计。
- 通用数据表。
- 语义框架。
- Agent 受控查询。

如果 BusinessGraphEdge 只完成基础存储而未完成多跳路径查询，可以进入简单业务场景验证，但不建议验证跨对象关系复杂的问题。

## 22. 后续衔接

基础平台 PRD 确认后，后续可以进入三类工作。

第一，原型设计。

围绕数据源、接入任务、数据表、数据目录、质量、权限、语义、图谱和 Agent 数据服务设计页面原型。

第二，业务场景验证方案。

选择一个高价值业务主线，例如订单交付风险、采购缺料、生产进度、库存异常或考勤人效，验证基础平台能力是否能支撑真实业务问题。

第三，研发实现计划。

在 PRD 和原型确认后，再拆解研发任务、接口、数据模型、权限模型、工具调用和验收用例。
